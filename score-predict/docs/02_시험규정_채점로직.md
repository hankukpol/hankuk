## 3. 과락 기준 ⚠️ (경찰과 다름!)

소방은 **2중 과락 기준**을 적용합니다.

### 3-1. 과목별 과락 (경찰과 동일)
- 각 과목에서 **과목 만점의 40% 미만** 득점 시 과락
  - 공채 각 과목 (만점 100점): **40점 미만** → 과락
  - 경채 소방학개론 (만점 100점): **40점 미만** → 과락
  - 경채 소방관계법규/응급처치학개론 (만점 100점): **40점 미만** → 과락

### 3-2. 총점 과락 (소방 전용! 경찰에는 없음)
- 전 과목 총점이 **총 만점의 60% 미만** 시 불합격
  - 공채: 300점 × 60% = **180점 미만** → 불합격
  - 경채: 200점 × 60% = **120점 미만** → 불합격

### 코드 구현 위치
- `src/lib/policy.ts`: `SUBJECT_CUTOFF_RATE = 0.4`, `TOTAL_CUTOFF_RATE = 0.6`
- `src/lib/scoring.ts`: `scoreByAnswerMap()` 함수에서 양쪽 과락 모두 체크

## 5. 가산점

### 5-1. 취업지원대상자 / 의사상자 가산점 (필기시험 단계 적용)

| 구분 | 가산 비율 | 제한 |
|------|---------|------|
| 취업지원대상자 | 과목별 만점의 **5% 또는 10%** | 선발인원 30% 초과 불가 |
| 의사상자 등 | 과목별 만점의 **3% 또는 5%** | 선발인원 10% 초과 불가 |

- 둘 중 1개만 선택 적용 (동시 적용 불가)
- **과목별 만점 기준**으로 산출 (문항당 점수 × 문항수가 아님)
- 가점은 과락 미해당 과목에만 적용됨

**DB 저장**: `Submission.bonusType` (BonusType enum: NONE/VETERAN_5/VETERAN_10/HERO_3/HERO_5)

### 5-2. 자격증 가산점 (최종 환산점수 적용 — 소방 전용!)

| 가산점 | 해당 자격 |
|-------|---------|
| 0점 | 해당 없음 |
| 2점 | 관련 자격증 소지 |
| 4점 | 관련 자격증 소지 |
| 5점 | 관련 자격증 소지 |

> **중요**: 자격증 가산점은 **필기시험 점수에 합산하지 않음**.
> 최종 환산점수(필기 50% + 체력 25% + 면접 25%)에 더해짐.
> 현재 서비스는 필기만 다루므로, 자격증 가산점은 참고용으로만 수집.

**DB 저장**: `Submission.certificateBonus` (Float, 0/2/4/5)

## 6. 성별 분리 선발 — 핵심 규칙 ★★★

> **석차·1배수·합격배수·합격예측은 반드시 아래 성별 기준 모집단 안에서만 계산합니다.**
> 성별 경계를 넘어 점수를 비교하거나 순위를 섞으면 절대 안 됩니다.

### 유형별 성별 기준

| ExamType | 응시자 | 비교 대상 | 모집인원 필드 |
|----------|------|---------|------------|
| `PUBLIC` 남자 | 공채 남자 | **공채 남자끼리만** | `recruitPublicMale` |
| `PUBLIC` 여자 | 공채 여자 | **공채 여자끼리만** | `recruitPublicFemale` |
| `CAREER_RESCUE` | 남자만 채용 | **단일 모집단** (여자 접수 불가) | `recruitRescue` |
| `CAREER_ACADEMIC` (양성 지역) | 남녀 모두 | **양성 통합 모집단** | `recruitAcademicCombined` |
| `CAREER_ACADEMIC` (분리 지역) 남자 | 소방학과 남자 | **소방학과 남자끼리만** | `recruitAcademicMale` |
| `CAREER_ACADEMIC` (분리 지역) 여자 | 소방학과 여자 | **소방학과 여자끼리만** | `recruitAcademicFemale` |
| `CAREER_EMT` 남자 | 구급 남자 | **구급 남자끼리만** | `recruitEmtMale` |
| `CAREER_EMT` 여자 | 구급 여자 | **구급 여자끼리만** | `recruitEmtFemale` |

### 이 규칙이 적용되는 계산 목록

- **석차**: 본인 성별 모집단 내 finalScore 내림차순 순위
- **1배수 커트 점수**: 본인 성별 모집인원 번째 사람의 점수
- **합격배수**: 본인 성별 모집인원 기준 N배수 계산
- **합격예측 피라미드**: 확실/유력/가능/도전 — 본인 성별 모집단 기준
- **경쟁자 목록**: 본인 성별 모집단의 타 참여자만 표시
- **평균점수/백분위**: 본인 성별 모집단 기준 통계

### 소방학과 (CAREER_ACADEMIC) 지역별 분기

같은 `CAREER_ACADEMIC`이라도 지역별로 채용 방식이 다릅니다:

```
if (quota.recruitAcademicCombined > 0) {
  → 양성 채용: 남녀 모두 같은 모집단, 모집인원 = recruitAcademicCombined
} else {
  → 분리 채용: 남자는 남자끼리 (recruitAcademicMale), 여자는 여자끼리 (recruitAcademicFemale)
}
```

**코드 구현**: `src/lib/prediction.ts`의 `buildPopulationWhere()`가 quota를 받아 분기 처리

> **전체 상세 규칙**: `docs/00_소방시험_전체구조_이해.md` 섹션 8 참고

### 양성(소방학과 경채) — 전형 단계별 선발 흐름

`CAREER_ACADEMIC`에서 `recruitAcademicCombined > 0`인 지역(양성 채용)의 **전체 전형 흐름**:

```
[필기시험]
  ├─ 과목별 과락 체크: 각 과목 40점 미만 → 탈락
  ├─ 총점 과락 체크: 120점 미만(200점 만점 × 60%) → 탈락
  └─ 합격배수 적용: 모집인원 × 배수(경채 기준) 이내 고득점자 → 필기 합격
        ※ 남녀 구분 없이 같은 점수 기준으로 통합 경쟁
        ※ 동점자는 전원 합격

      ↓ 필기 합격자

[체력시험]
  └─ 기준 통과자만 다음 단계 진출
        ※ 성별 불문 동일 기준 적용 (양성 통합)

      ↓ 체력 합격자

[면접]
  └─ 면접 점수 산출

      ↓

[최종 선발]
  최종점수 = 필기(50%) + 체력(25%) + 면접(25%)
  → 최종점수 고득점 순으로 모집인원만큼 선발
  → 자격증 가산점 있으면 최종점수에 추가 반영
  → 남녀 구분 없이 통합 고득점 순 선발 (양성 채용의 핵심)
```

> **서비스 범위**: 현재는 **필기 단계**만 예측합니다.
> 체력·면접·최종환산 계산은 미구현이며, 자격증 가산점(`certificateBonus`)은 DB에 저장만 하고 필기 순위 계산에 반영하지 않습니다.

## 9. 수험번호 체계 ★★★

소방 수험번호는 **총 10자리 숫자**로 구성되며, 자리별로 **성별·채용유형 정보**가 인코딩됩니다.

### 9-1. 수험번호 자릿수 구조

```
2  1  1  [성별]  [유형코드]  [순번 4자리]
─────────────────────────────────────────
자리: 1  2  3    4       5~6      7~10
```

| 자리 | 의미 | 값 |
|------|------|-----|
| 1~3번째 | 지역/시험 식별 prefix | 예: `211` (지역마다 상이할 수 있음) |
| 4번째 | **성별 코드** | 남자=`1`, 여자=`2` |
| 5~6번째 | **채용유형 코드** | 아래 표 참고 |
| 7~10번째 | **순번** | `0001`부터 시작, 4자리 0패딩 |

### 9-2. 채용유형 코드

| 코드 | 채용분야 | 성별 |
|------|---------|------|
| `01` | 공채 | 남/여 분리 |
| `03` | 구급 경채 | 남/여 분리 |
| `04` | 구조 경채 | 남자만 (`04`는 항상 성별코드 `1`) |
| `05` | 소방관련학과 경채 | 남/여 분리 (양성 지역도 동일 코드) |

### 9-3. 수험번호 범위 예시 (이미지 기준)

> 아래는 특정 지역의 실제 배부 예시입니다. prefix `211`은 지역마다 다를 수 있습니다.

| 채용구분 | 채용분야 | 수험번호 범위 | 응시인원 |
|---------|---------|-------------|---------|
| 공개경쟁채용 | 공채(남) | 2111010001 ~ 2111010745 | 745명 |
| 공개경쟁채용 | 공채(여) | 2112010001 ~ 2112010143 | 143명 |
| 경력경쟁채용 | 구급(남) | 2111030001 ~ 2111030152 | 152명 |
| 경력경쟁채용 | 구급(여) | 2112030001 ~ 2112030112 | 112명 |
| 경력경쟁채용 | 구조 | 2111040001 ~ 2111040065 | 65명 |
| 경력경쟁채용 | 소방관련학과(남) | 2111050001 ~ 2111050056 | 56명 |
| 경력경쟁채용 | 소방관련학과(여) | 2112050001 ~ 2112050027 | 27명 |

### 9-4. 수험번호 파싱 예시

```
2111030152 → 211 | 1 | 03 | 0152
              ↑    ↑    ↑    ↑
            prefix 남  구급  152번째
```

```
2112050027 → 211 | 2 | 05 | 0027
              ↑    ↑    ↑    ↑
            prefix 여  학과  27번째
```

### 9-5. DB 연관 필드

`ExamRegionQuota` 모델에 지역별 수험번호 범위를 저장합니다:

```prisma
examNumberStart  String?  // 예: "2111010001"
examNumberEnd    String?  // 예: "2111010745"
```

> **주의**: 현재 `examNumberStart/End`는 공채 기준으로만 설계되어 있습니다.
> 경채 유형별 범위는 별도 필드가 없으므로, 수험번호 자릿수 구조로 유형 파악이 가능합니다.

### 9-6. 수험번호 입력 검증 규칙

- **형식**: 10자리 숫자 문자열
- **유형 일치**: 4번째 자리(성별코드)가 제출 시 선택한 성별과 일치해야 함
  - 단, 현재 서비스에서는 강제 검증하지 않고 입력 받음 (사용자 실수 가능성 고려)
- **범위**: `examNumberStart ~ examNumberEnd`가 설정된 경우 범위 이탈 시 제출 차단
  (→ `submission/route.ts`의 `isExamNumberOutOfRange()` 함수)

## 2026-03 수험번호 검증 업데이트

### 응시번호 범위 필드 (신규)

`ExamRegionQuota` 신규 필드:

```prisma
examNumberStartPublicMale              String?
examNumberEndPublicMale                String?
examNumberStartPublicFemale            String?
examNumberEndPublicFemale              String?
examNumberStartCareerRescue            String?
examNumberEndCareerRescue              String?
examNumberStartCareerAcademicMale      String?
examNumberEndCareerAcademicMale        String?
examNumberStartCareerAcademicFemale    String?
examNumberEndCareerAcademicFemale      String?
examNumberStartCareerAcademicCombined  String?
examNumberEndCareerAcademicCombined    String?
examNumberStartCareerEmtMale           String?
examNumberEndCareerEmtMale             String?
examNumberStartCareerEmtFemale         String?
examNumberEndCareerEmtFemale           String?
```

기존 `examNumberStart`, `examNumberEnd`는 deprecated 하위호환 필드로 유지한다.

### 저장/검증 규칙

- 수험번호 형식: `^\\d{10}$`
- 자리 규칙: `1~3 prefix`, `4 성별코드(1/2)`, `5~6 유형코드`, `7~10 순번`
- 유형코드: `01 공채`, `03 구급`, `04 구조`, `05 학과`
- 구조는 남자만 허용(성별코드 `1`)
- 학과 양성(통합 경쟁)은 성별코드 `1/2` 모두 허용
- 범위는 직렬/성별(또는 양성 통합)별 필드 기준으로만 비교
- 관리자 저장 시 범위는 `둘 다 비움` 또는 `둘 다 10자리 입력`만 허용
- 관리자 저장 시 `start <= end`를 강제
- 관리자 저장 시 시작/끝 값의 4,5~6자리 코드가 탭 규칙과 일치해야 저장 가능
