# PRD: 출결관리 시스템 + 문서 출력 + 공무원학원 특화 기능

**작성일**: 2026-03-13
**포함 요청**:
- 학생 강의 출결관리 시스템 (신규)
- 수강내역 확인서 등 문서 출력
- 공무원학원 특화 기능 조사 및 추가 제안

---

## 현재 시스템의 출결 vs 신규 출결의 차이

| 구분 | 현재 시스템 | 신규 필요 |
|---|---|---|
| **대상** | 아침 모의고사 응시 여부 | **강의 수업** 출결 |
| **단위** | 시험 회차별 (과목별) | 수업 시간별 (시간표 기반) |
| **상태** | NORMAL/LIVE/EXCUSED/ABSENT | 출석/지각/조퇴/결석/공결 |
| **체크 방법** | 성적 입력 시 자동 | QR/수기/카카오 파싱 |
| **법적 의미** | 내부 관리용 | **학원법 출결 기준** (환불 산정) |

> 두 시스템은 **별도로 운영**한다. 아침모의고사 출결은 현행 유지.
> 강의 출결은 신규 모듈로 추가.

## PART 1. 강의 출결관리 시스템

### 1-1. 개념 구조

```
강좌 (Course)
  └─ 수업 일정 (LectureSchedule)  ← 요일/시간/강의실 정의
       └─ 수업 회차 (LectureSession)  ← 실제 진행된 수업 1건
            └─ 출결 기록 (LectureAttendance)  ← 학생별 출결 상태
```

### 1-2. 출결 상태 5종

| 상태 | 코드 | 설명 | 출석 인정 |
|---|---|---|:---:|
| 출석 | PRESENT | 정상 출석 | ✓ |
| 지각 | LATE | 수업 시작 후 입실 | ✓ (0.5회) |
| 조퇴 | EARLY_LEAVE | 수업 중간 퇴실 | ✓ (0.5회) |
| 공결 | EXCUSED | 사유서 승인된 결석 | ✓ |
| 결석 | ABSENT | 무단 결석 | ✗ |

### 1-3. 시간표 관리

#### DB 모델

```prisma
// 강의 시간표 (반복 일정 정의)
model LectureSchedule {
  id            Int        @id @default(autoincrement())
  courseId      Int
  classroomId   Int?       // 반 배정 (null이면 강좌 전체)
  dayOfWeek     Int        // 0=일, 1=월 ... 6=토
  startTime     String     // "09:00"
  endTime       String     // "11:00"
  teacherStaffId String?   // 담당 강사
  room          String?    // 강의실 ("301호")
  validFrom     DateTime   // 시간표 적용 시작일
  validUntil    DateTime?  // 시간표 종료일 (null=무기한)
  isActive      Boolean    @default(true)
}

// 실제 수업 회차 (진행된 수업 1건)
model LectureSession {
  id             String     @id @default(cuid())
  scheduleId     Int
  courseId       Int
  sessionDate    DateTime   // 수업 날짜
  startTime      String     // 실제 시작 시간
  endTime        String?    // 실제 종료 시간
  topic          String?    // 수업 주제 (강사 메모)
  isCancelled    Boolean    @default(false)
  cancelReason   String?
  staffId        String     // 입력 직원
  createdAt      DateTime   @default(now())

  attendances    LectureAttendance[]
}

// 학생별 출결
model LectureAttendance {
  id             String     @id @default(cuid())
  sessionId      String
  examNumber     String
  status         LectureAttendStatus  // PRESENT/LATE/EARLY_LEAVE/EXCUSED/ABSENT
  checkinTime    String?    // 실제 입실 시간 ("09:07")
  checkoutTime   String?    // 퇴실 시간 (조퇴 시)
  note           String?
  source         LectureAttendSource // QR/MANUAL/KAKAO
  staffId        String?
  createdAt      DateTime   @default(now())

  @@unique([sessionId, examNumber])
}

enum LectureAttendStatus { PRESENT LATE EARLY_LEAVE EXCUSED ABSENT }
enum LectureAttendSource { QR MANUAL KAKAO BULK }
```

### 1-4. 출결 체크 방식 3가지

#### A. QR코드 체크인

- 학생이 스마트폰으로 강의실 QR 스캔
- QR 코드는 수업마다 자동 갱신 (수업 시작 전 5분 ~ 종료 시)
- 스캔 시간 기준으로 출석/지각 자동 판정
- 강의실 입구에 A4 QR 출력 또는 TV 화면 표시

구현: `/student/check-in?sessionId=xxx` → 학생 포털 URL
QR 생성: `qrcode` npm 패키지 활용

#### B. 수기 일괄 입력

- 관리자/강사가 출석부 리스트 보면서 클릭으로 입력
- 기본값: 전체 출석 → 결석/지각만 변경
- 반별 학생 목록이 자동 표시

#### C. 카카오톡 파싱 연동

기존 담임 PRD의 카카오 파싱 기능을 강의 출결에도 적용.

### 1-6. 학원법 출결 기준 자동 계산

학원의 설립·운영 및 과외교습에 관한 법률에 따른 수강료 환불 기준:

```
총 수업 횟수 기준 출석률 계산
환불 금액 = 미수강분 × 일할 계산

시스템 자동 계산 항목:
- 등록일 ~ 오늘 진행된 수업 횟수
- 실제 출석 횟수 (공결 포함)
- 잔여 수업 횟수
- 환불 가능 금액 (학원법 기준)
```

출결 기반 환불 계산 → 수납 모듈의 환불 처리와 연동.

#### 구현 방법

```
/admin/members/[id]/documents/enrollment-certificate
  → 인쇄 미리보기 페이지 (print: 전용 CSS로 A4 레이아웃)
  → window.print() 또는 PDF 다운로드

URL 파라미터:
  ?from=2026-01-01&to=2026-03-13  ← 기간 선택

**인쇄 CSS** (`@media print`):
- 용지: A4, 세로
- 여백: 상하 20mm, 좌우 25mm
- 원장 직인 이미지 업로드 기능 (Settings에서 등록)

## PART 3. 공무원학원 특화 기능 제안

입시학원과 구분되는 **공무원학원만의 특성**에서 도출한 기능들.

### 3-1. 공무원 시험 일정 관리

공무원 수험생은 실제 시험 일정에 맞춰 공부한다. 시험 일정을 관리하고 학생과 연결한다.

#### A. 시험 캘린더

```
/admin/exam-schedule                공무원 시험 일정 관리

2026년 공채 시험 일정
┌────────────────────────────────────────────────┐
│ 시험명              필기      면접      발표     │
│ ────────────────────────────────────────────   │
│ 경찰공무원(순경) 1차  04/26    06/05    07/15   │
│ 경찰공무원(순경) 2차  08/22    10/10    11/20   │
│ 경찰간부 필기         03/14    -        -       │
└────────────────────────────────────────────────┘
```

#### B. 시험 D-Day 연동

- 대시보드에 "경찰공무원 필기 D-42" 카운트다운 표시
- 학생 포털에도 본인 목표 시험 D-Day 표시
- 시험 2주 전 → 자동 격려 SMS 발송 옵션

#### C. 학생별 목표 시험 설정

학생 상세 페이지에서 응시 예정 시험 등록:
- 목표 시험명, 응시 예정 회차
- 필기 합격 여부 → 면접 대비 모드 전환
- 최종 합격 여부 → 합격자 등록

### 3-2. 합격자 관리

공무원학원의 핵심 실적 지표 = **합격률**.

#### A. 합격 보고

```
합격자 등록

### 3-3. 재수강생 (N수생) 관리

공무원 수험생의 특성: 합격할 때까지 수년간 반복 수강.

#### A. 수강 이력 연속성

```
홍길동 수강 이력
  2023 1기 (7개월) → 불합격
  2024 1기 (7개월) → 필기합격 → 면접 불합격
  2025 1기 (현재 수강중)
```

- 기수 변경 시에도 동일 수험번호로 이력 연속 유지
- 기수별 성적 비교 차트 (현재 `CumulativeAnalysisData`에 이미 구현됨)
- 수강 총 기간 자동 계산 (몇 년 몇 개월 수강중)

#### B. 장기수강생 관리

- N기 이상 수강생 자동 플래그 (예: 3기 이상)
- 장기수강생 면담 주기 알림 (매월 1회 의무 면담 등)
- 장기수강생 특별 할인 적용 옵션

### 3-4. 자습실 이용 관리

독서실 형태 자습실을 운영하는 공무원학원에 필요한 기능.

#### A. 자습실 좌석 관리

```
자습실 현황 (2층, 총 80석)

오늘 입실: 60명  누적 이용: 4시간 23분(평균)
```

#### B. 자습실 기능

### 3-5. 특강/이벤트 관리

#### A. 특별 강좌

| 기능 | 설명 |
|---|---|
| 특강 개설 | 기간, 대상(공채/경채/전체), 강사, 수강료 |

#### B. 시험 전 특별 관리 기간

시험 D-30일 진입 시 자동 전환되는 집중 관리 모드:
- 일일 모의고사 횟수 증가
- 면담 주기 단축 (2주 → 1주)
- 학생 포털에 특별 일정 공지

### 3-6. 강사 관리

## PART 4. 전체 기능 보완 목록 (추가 제안)

현재 계획에 없지만 공무원학원 운영에 실용적인 기능들.

### 4-1. 학습 목표 관리

```
/admin/students/[examNumber] → 학습 목표 탭

주차별 학습 목표 설정
  3월 3주: 형법 10회독 완성, 경찰학 모의고사 75점 달성
  달성 여부: [완료] [미완료] [부분완료]

누적 목표 달성률 차트
```

### 4-2. SMS/알림 고도화

| 알림 종류 | 발송 시점 |
|---|---|
| 수업 시작 알림 | 수업 30분 전 (학생) |
| 결석 알림 | 수업 시작 30분 후 미출석 시 (보호자) |
| 수강료 납부 독촉 | 납부일 D-3, D-1, 당일 |
| 시험 D-Day 알림 | D-30, D-14, D-7, D-1 |
| 합격 축하 | 합격 등록 시 자동 |
| 면담 예약 확인 | 면담 전날 |

### 4-3. 학원 운영 보고서

원장이 매월 볼 수 있는 종합 리포트 (`/admin/reports/monthly`):

미수금: 2,100,000원 (3명)
환불: 600,000원 (1명)

[ 출결 현황 ]

개근자: 34명 (39.1%)

아침 모의고사 전체 평균: 68.4점

```

### 4-4. 민원/건의 관리

학생 문의, 불만, 건의사항을 시스템에서 관리.

```
/admin/tickets

민원 유형: 환불, 성적이의, 시설, 수업, 기타
처리 상태: 접수 → 처리중 → 완료
답변 등록 → 학생에게 SMS 발송
```

### 4-5. 강의 영상 링크 관리

강의 녹화 영상을 학생에게 제공하는 경우:

```

학생 조회 가능 기간 설정
조회 횟수 통계
```

## PART 5. 개발 우선순위 (통합 로드맵 업데이트)

기존 마스터플랜에 이번 PRD 내용 추가 후 우선순위 재정리:

```
Phase 0 (기반 작업)

  기존 내용 + 문서 출력 기능 (수강내역 확인서, 수납 영수증)

  P2-NEW-1: 시간표 관리 (LectureSchedule)

  P2-NEW-3: QR 체크인 시스템
  P2-NEW-4: 출결 현황판 (오늘 수업)
  P2-NEW-5: 학생별 출결 월별 집계 + 출결 확인서 출력

Phase 3 (상담·단과 POS)
  기존 내용 유지

Phase 4 (시설 관리)
  기존 내용 + 자습실 관리 추가

Phase 5 (공무원 특화)
  P5-SPEC-1: 공무원 시험 일정 캘린더
  P5-SPEC-2: 합격자 관리 + 합격 현황판
  P5-SPEC-3: 재수강생 수강 이력 연속성
  P5-SPEC-4: 장기수강생 자동 플래그 + 면담 알림

Phase 6 (성적 통합)
  기존 내용 + 성적 통지표 출력

Phase 7 (고도화)
  세금계산서·현금영수증 API
  강사 수당 계산
  월간 운영 보고서 자동 생성

  학원법 기준 환불 자동 계산

## PART 6. 신규 DB 모델 요약 (이번 PRD 추가분)

```prisma
// 강의 출결
model LectureSchedule { ... }     // 시간표
model LectureSession { ... }      // 수업 회차
model LectureAttendance { ... }   // 학생별 출결

// 문서
model DocumentLog {               // 문서 발행 이력
  id          String    @id @default(cuid())
  examNumber  String
  docType     DocType   // ENROLLMENT_CERT / ATTEND_CERT / RECEIPT / REPORT_CARD
  issuedAt    DateTime  @default(now())
  issuedBy    String    // 발행 직원
  note        String?
}

// 공무원 시험
model CivilServiceExam {          // 시험 일정
  id          Int       @id @default(autoincrement())
  name        String
  examType    ExamType
  writtenDate DateTime?
  interviewDate DateTime?
  resultDate  DateTime?
  isActive    Boolean   @default(true)
}

model StudentExamTarget {         // 학생별 목표 시험
  examNumber  String
  examId      Int
  status      ExamTargetStatus  // PREPARING/WRITTEN_PASS/FINAL_PASS/FAILED
  setAt       DateTime @default(now())
}

// 합격자
model GraduateRecord {
  id          String    @id @default(cuid())
  examNumber  String
  examId      Int
  passType    PassType  // WRITTEN / FINAL
  passedAt    DateTime
  note        String?
  staffId     String
  createdAt   DateTime  @default(now())
}

// 학원 기본 설정
model AcademySettings {
  id              Int     @id @default(1)
  name            String  // 학원명
  directorName    String  // 원장명
  businessNumber  String? // 사업자등록번호
  registrationNo  String? // 학원등록번호
  address         String?
  phone           String?
  logoUrl         String?
  sealUrl         String? // 원장 직인 이미지
  updatedAt       DateTime @updatedAt
}

enum DocType         { ENROLLMENT_CERT ATTEND_CERT RECEIPT REPORT_CARD }
enum ExamTargetStatus{ PREPARING WRITTEN_PASS FINAL_PASS FAILED }
enum PassType        { WRITTEN FINAL }
```

## ⚠️ 멀티지점 고려사항 (2026-03-21 추가)

### 데이터 격리
- 모든 출결 데이터 (Attendance, AbsenceNote)는 `academyId`로 격리됨
- 지점 관리자는 자신의 지점 출결 데이터만 접근 가능

### 설정 독립화
- 결석 허용 횟수: 지점별 독립 (`/admin/settings/absence-policies`)
- 자동 경고 발송 기준: 지점별 독립
- 결석계 제출 마감 기한: 지점별 독립

### 개발 시 주의사항
- AbsenceNote 조회: `where: { academyId: ctx.academyId }` 필터 적용
- 출결 통계: 지점별 집계
