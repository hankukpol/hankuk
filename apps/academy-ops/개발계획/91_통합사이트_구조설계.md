# 통합 학원 관리 시스템 — 사이트 구조 설계

**작성일**: 2026-03-13
**목적**: 다양한 기능 모듈을 하나의 통합 사이트로 구축하기 위한 URL 구조, 네비게이션, 권한 설계

---

## 1. 핵심 원칙: 단일 앱 (Monolith)

모든 기능을 **하나의 Next.js 14 앱**으로 구축한다.
분리된 서비스(마이크로서비스)로 나누지 않는다.

### 이유

| 항목 | 단일 앱 | 서비스 분리 |
|---|---|---|
| 학생 데이터 공유 | 그냥 DB 조회 | API 통신 필요 |
| 권한 체계 통합 | 하나의 미들웨어 | 각 서비스마다 별도 구현 |
| 개발 속도 | 빠름 | 느림 |
| 운영 비용 | 낮음 (Vercel 1개) | 높음 |
| 현재 기술 스택 유지 | 완전 호환 | 마이그레이션 필요 |

## 5. 구현 시 핵심 기술 결정사항

### 5-1. 현재 `Student` vs 신규 `AcademyMember`

```
현재: Student.examNumber (수험번호 = PK)
신규: AcademyMember.id  (자동증가 숫자 ID = PK)
         ↕ examNumber FK (nullable — 수험번호 없는 상담 접수자도 있음)
```

수험번호가 없는 상담 문의자도 시스템에 등록할 수 있어야 하므로, 신규 회원 마스터는 별도 ID를 사용한다.

### 5-2. 레이아웃 분리

```
/admin/layout.tsx   ← 좌측 사이드바 포함 (현재: 없음 → 신규 생성 필요)
/student/layout.tsx ← 학생 포털 헤더 (현재 존재)
```

현재 `/admin`에는 layout.tsx가 없고 각 페이지에 네비게이션이 인라인으로 들어가 있다. 통합 사이드바 도입 시 `admin/layout.tsx`를 새로 만들어야 한다.

### 5-3. 단계적 통합 전략

```
Phase 0: 기존 시스템 유지
  → 사이드바 layout.tsx 추가 (기존 ADMIN_NAV_ITEMS 활용)
  → 신규 모듈 메뉴 항목 사전 추가 (비활성 상태)

Phase 1~N: 모듈별 개발
  → 각 모듈 완성 시 해당 메뉴 활성화
  → 기존 페이지는 그대로 유지, URL 충돌 없음
```

## 6. 멀티지점 URL 구조 추가 (2026-03-21)

> 단일 코드베이스에서 복수 지점 운영. URL 변경 없음 (세션 컨텍스트로 지점 구분).

### 6-1. 슈퍼관리자 전용 URL

```
/admin/super/                      ← SUPER_ADMIN 전용 영역 (별도 auth 체크)
├── /admin/super/academies          전체 지점 목록 + 생성 + 활성화/비활성화
├── /admin/super/academies/new      신규 지점 등록
├── /admin/super/academies/[id]     지점 상세·수정
├── /admin/super/dashboard          전 지점 통합 KPI 대시보드
└── /admin/super/users              전 지점 관리자 계정 관리
```

### 6-2. 신규 설정 URL

```
/admin/settings/
├── exam-subjects     ← 지점별 시험 과목 동적 관리 (신규)
├── scoring-rules     ← 지점별 성적 처리 규칙 (신규)
└── ... (기존 유지)
```

### 6-3. 지점 전환 API

```
POST /api/admin/switch-academy      지점 전환 (쿠키 activeAcademyId 설정)
GET  /api/academies                  전체 지점 목록 (SUPER_ADMIN 전용)
POST /api/academies                  신규 지점 생성 (SUPER_ADMIN 전용)
```

### 6-4. 탭 구조 변경 (6탭 → 7탭)

```
[기존]
대시보드 | 수강관리 | 수납·결제 | 성적관리 | 출결관리 | 시설관리 | 시스템

[변경 후]
대시보드 | 회원·수강 | 수납·결제 | 성적·출결 | 시설·포인트 | 분석·보고 | 설정
```
- 성적 + 출결 통합 → "성적·출결"
- analytics 독립 분리 → "분석·보고"
- 포인트 시설 탭으로 이동 → "시설·포인트"
- 설정 탭에 슈퍼관리자 섹션 추가
