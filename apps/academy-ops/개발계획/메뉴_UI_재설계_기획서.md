# 메뉴 구조 및 UI 재설계 기획서
## 한국경찰학원 통합 관리 시스템 — 벤치마크 기반 최적화 제안

**작성일**: 2026-03-21
**참고**: 에듀그램, 클래스업, Jackrabbit Class, iClassPro, 공선학관 등 국내외 사례 분석 기반

---

## 핵심 설계 원칙

> **원칙 1**: 자주 쓰는 기능 → 최소 클릭으로 접근
> **원칙 2**: 각 페이지는 하나의 목적만 수행
> **원칙 3**: 데이터 흐름 (학생등록 → 수강등록 → 수납)이 메뉴 순서와 일치
> **원칙 4**: 관리(운영) / 분석(조회) / 설정(세팅)을 명확히 분리

## 1. 상단 모듈 탭 재편 (현재 6개 → 7개)

### 현재 vs 제안

| 현재 탭 | 문제점 | 제안 탭 |
|---------|--------|---------|
| 대시보드 | - | ✅ **대시보드** |
| 수강관리 | 결제 항목 6개 혼재 | ✅ **회원·수강** |
| 수납·결제 | 일부 항목 누락 | ✅ **수납·결제** |
| 성적관리 | analytics 25개가 성적관리에 포함 | ✅ **성적·출결** (출결 통합) |
| 출결관리 | 성적과 연관성 높아 분리 불필요 | ❌ 성적과 통합 |
| 시설관리 | 포인트가 시설에 부자연스럽게 포함 | ✅ **시설·포인트** |
| (없음) | analytics가 성적 모듈에 숨겨짐 | ✅ **분석·보고** (신규) |
| 시스템 | 운영마스터와 시스템설정 혼재 | ✅ **설정** |

### 제안 탭 구조 (7개)

```
[ 대시보드 ] [ 회원·수강 ] [ 수납·결제 ] [ 성적·출결 ] [ 시설·포인트 ] [ 분석·보고 ] [ 설정 ]
```

## 3. 페이지 UI 레이아웃 표준

### 3-1. 전체 레이아웃 구조

```
┌────────────────────────────────────────────────────────┐
│  [로고] [대시보드][회원·수강][수납·결제][성적·출결]...  │  ← 상단 모듈 탭 (56px)
├──────────┬─────────────────────────────────────────────┤
│          │  페이지 제목                  [액션 버튼]   │  ← 페이지 헤더 (48px)
│  사이드  ├─────────────────────────────────────────────┤
│  바      │                                             │
│  (224px) │  메인 컨텐츠 영역                           │
│          │                                             │
│  [그룹1] │                                             │
│  - 메뉴1 │                                             │
│  - 메뉴2 │                                             │
│          │                                             │
│  [그룹2] │                                             │
│  - 메뉴3 │                                             │
└──────────┴─────────────────────────────────────────────┘
```

### 3-2. 페이지 유형별 UI 레이아웃

---

#### 유형 A: 대시보드 페이지 (원장/관리자 홈)

```
┌──────────────────────────────────────────────────────────┐
│ 페이지 제목: 오늘 현황 (날짜)                             │
├──────────┬──────────┬──────────┬──────────┐
│ KPI 카드 │ KPI 카드 │ KPI 카드 │ KPI 카드 │  ← 4개 핵심 지표
│ 총 수강생 │ 오늘 출석 │ 이번달수납│ 미납현황 │    (카드 높이 120px)
├──────────┴──────────┴──────────┴──────────┘
│                                                          │
│  ┌─────────────────────┐  ┌──────────────────────────┐  │
│  │ 최근 수납 내역       │  │ 미납 학생 (Top 5)        │  │
│  │ (5건, 더보기 링크)   │  │ (이름, 금액, 경과일)     │  │
│  └─────────────────────┘  └──────────────────────────┘  │
│                                                          │
│  ┌─────────────────────┐  ┌──────────────────────────┐  │
│  │ 이번 주 수강 만료    │  │ 최근 상담 방문자          │  │
│  │ (학생명, 만료일)     │  │ (이름, 방문일, 단계)      │  │
│  └─────────────────────┘  └──────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

**UI 규칙:**
- KPI 카드 4개 → 숫자 크게 (2rem), 부제목 작게 (0.75rem)
- 카드 클릭 시 해당 상세 페이지 이동
- 하단 위젯은 최대 4개, 각 5~7건만 표시 + "더보기" 링크

---

#### 유형 B: 목록(List) 페이지

```
┌──────────────────────────────────────────────────────────┐
│ 페이지 제목: 전체 학생 명단           [+ 신규 등록]      │  ← 헤더 (우측: 주요 액션 1개)
├──────────────────────────────────────────────────────────┤
│ [검색창         🔍]  [기수▼] [상태▼] [직렬▼]  [초기화] │  ← 필터 바 (1줄)
├──────────────────────────────────────────────────────────┤
│ 총 254명 (공채 198 / 경채 56)                            │  ← 요약 카운트
├───────┬──────┬──────┬──────┬───────┬────────────────────┤
│ 학번  │ 이름 │ 연락처│ 직렬 │ 상태  │ 액션               │  ← 테이블 헤더
├───────┼──────┼──────┼──────┼───────┼────────────────────┤
│ 24001 │ 홍길동│010...│ 공채 │ 재원중 │ [상세] [수강]      │
│ 24002 │ 김영희│010...│ 경채 │ 경고1  │ [상세] [수강]      │
├───────┴──────┴──────┴──────┴───────┴────────────────────┤
│                      [< 1 2 3 ... >]                     │  ← 페이지네이션
└──────────────────────────────────────────────────────────┘
```

**UI 규칙:**
- 헤더 우측에 주요 액션 버튼 최대 2개 (ember 색)
- 필터는 1줄, 접는 버튼으로 고급 필터 제공
- 테이블 컬럼: 5~7개 (너무 많으면 가로 스크롤)
- 행 클릭 → 상세 페이지 이동 (클릭 영역 전체)
- 액션 버튼은 아이콘+텍스트 또는 아이콘만 (행당 최대 2개)
- 페이지당 20~50건 표시 (기본 30건)

#### 유형 C: 상세(Detail) 페이지 — 학생 프로필

```
┌──────────────────────────────────────────────────────────┐
│ ← 뒤로   학생 상세: 홍길동 (24001)   [수강등록] [수납]  │  ← 헤더
├──────────────────────────────────────────────────────────┤
│ ┌──────────────────────────────────────────────────────┐ │
│ │ 사진  학번: 24001  이름: 홍길동  연락처: 010-xxxx    │ │  ← 학생 기본정보 카드
│ │       직렬: 공채   상태: 🟢 재원중  생년월일: 99.01.01│ │    (항상 상단 고정)
│ └──────────────────────────────────────────────────────┘ │
├─────────┬────────┬────────┬────────┬────────┬────────────┤
│ 수강이력 │ 수납이력│ 성적현황│ 출결현황│  면담  │   메모     │  ← 탭 (6개 이하)
├──────────────────────────────────────────────────────────┤
│                                                          │
│  [선택된 탭 내용]                                        │
│                                                          │
│  수강이력 탭 예시:                                        │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 2024 상반기 종합반   ACTIVE    2024.01~2024.06   │   │
│  │ 2023 하반기 종합반   COMPLETED 2023.07~2023.12   │   │
│  └──────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

**UI 규칙:**
- 상단 학생 기본정보 카드 = 항상 표시 (스크롤해도 고정 또는 sticky)
- 탭: 최대 6개, 탭 제목 짧게 (4글자 이하)
- 탭 내 데이터는 **읽기 전용** 위주 + "바로가기" 버튼으로 전용 페이지 이동
- 우측 상단 버튼: 해당 학생에 대한 주요 액션 (수강등록, 수납등록)

#### 유형 D: 등록(Form) 페이지

```
┌──────────────────────────────────────────────────────────┐
│ ← 뒤로   수납 등록                                       │  ← 헤더 (저장 버튼 없음)
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 1단계: 학생 선택                                  │   │  ← 스텝 표시 (복잡한 폼)
│  │ 학번/이름 검색: [검색창          🔍]              │   │
│  │                                                  │   │
│  │ ✓ 홍길동 (24001) 공채 재원중                     │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 2단계: 수납 정보                                  │   │
│  │ 수납 항목  [수강료▼]    금액  [______원]          │   │
│  │ 결제 방법  [현금▼]      메모  [____________]      │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│              [취소]              [저장 · 영수증 출력]    │  ← 액션 버튼 (우측 하단)
└──────────────────────────────────────────────────────────┘
```

**UI 규칙:**
- 복잡한 폼(3단계 이상): 스텝 인디케이터 상단 표시
- 단순한 폼: 스텝 없이 섹션(제목+필드 그룹)으로 구분
- 필수 입력 필드: `*` 표시
- 저장 버튼: 우측 하단 (ember 색, 주요 버튼)
- 취소 버튼: 저장 버튼 왼쪽 (테두리형)
- 저장 후: 상세 페이지 또는 목록 페이지로 이동

## 5. 학생 상세 페이지 탭 재편

### 현재 (12탭) → 제안 (6탭)

| 현재 탭 | 문제 | 제안 탭 |
|---------|------|---------|
| 성적이력 | | ✅ **수강·성적** (통합) |
| 성적차트 | 탭으로 보기엔 과함 | → 수강·성적 탭 내 차트로 |
| 누적분석 | | → 수강·성적 탭 내 |
| 기간별분석 | | → 수강·성적 탭 내 |
| 타임라인 | | → 수강·성적 탭 내 |
| 면담 | | ✅ **출결·면담** (통합) |
| 수업 | | → 출결·면담 탭 내 |
| 수납 | | ✅ **수납·결제** |
| 포인트 | | → 수납·결제 탭 내 |
| 출결 | | → 출결·면담 탭 내 |
| 오답노트 | | ✅ **학습 자료** |
| 메모 | | ✅ **메모·이력** |

**제안: 6탭 구조**

```
[ 수강·성적 ] [ 출결·면담 ] [ 수납·결제 ] [ 학습 자료 ] [ 서류 ] [ 메모·이력 ]
```

각 탭 내에서 "전체 보기 →" 링크로 전용 관리 페이지 이동.

## 9. 구현 우선순위 (메뉴 재편 관점)

| 우선순위 | 작업 | 이유 |
|---------|------|------|
| **1순위** | 상단 탭에 "분석·보고" 독립 추가 | analytics 찾기 어려움 해소 |
| **2순위** | 성적·출결 탭 통합 (출결 탭 제거) | 두 기능 연관성 높음 |
| **3순위** | 수강관리 탭에서 수납 항목 6개 제거 | 모듈 경계 정리 |
| **4순위** | 학생 상세 12탭 → 6탭 통합 | 복잡도 감소 |
| **5순위** | 결제링크 페이지 단일화 | 두 곳 혼란 해소 |
| **6순위** | 설정 내 운영마스터 항목 해당 모듈로 이동 | 설정 경량화 |
