## 4. DB 스키마 관리

### 스키마 변경 시 (개발 중)

```bash
cd fire

# 스키마 수정 후 DB에 적용 (마이그레이션 없이 바로 push)
npx prisma db push

# Prisma Client 재생성
npx prisma generate
```

### 마이그레이션 파일 관리 (운영 배포 시)

```bash
# 마이그레이션 생성
npx prisma migrate dev --name "add_xxx_field"

# 운영 DB에 적용
npx prisma migrate deploy
```

## 5. 개발 명령어

```bash

npx prisma generate   # Prisma Client 재생성
```

## 6. 서버 실행 문제 해결

### 문제: `tailwindcss` 모듈을 부모 디렉토리에서 찾으려 함

**증상**: 서버가 로딩 중으로 계속 대기, 콘솔에 아래 에러
```
Error: Can't resolve 'tailwindcss' in 'D:\앱 프로그램\합격예측 프로그램'
```

**원인**: Turbopack이 CSS `@import "tailwindcss"` 경로를 부모 디렉토리에서 해석
**해결**:
```bash
# 방법 1: Turbopack 비활성화
npx next dev -p 3200 --no-turbo

# 방법 2: dev-start.bat 사용 (scripts/dev-single-process.cjs 통해 실행)
```

`package.json`의 dev 스크립트 수정 가능:
```json
"dev": "next dev -p 3200 --no-turbo"
```

### 문제: DB seed 실행 중 `P1017 Server has closed the connection`

**원인**: Transaction pooler(6543포트)는 장시간 연결 유지에 부적합
**해결**: Session pooler(5432포트)로 DATABASE_URL 직접 지정해서 실행
```bash
DATABASE_URL="postgresql://postgres.iqhkmcxeuwueiqopkwfd:hankukpol_0112%5E%5E@aws-1-ap-northeast-2.pooler.supabase.com:5432/postgres?sslmode=require" npx prisma db seed
```
