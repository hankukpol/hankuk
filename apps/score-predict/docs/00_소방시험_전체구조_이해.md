## 1. 소방공무원 채용 전체 흐름

```
[공고] → [접수] → [필기시험] → [체력시험] → [면접] → [최종합격]
                       ↑
               이 서비스가 다루는 구간
               (필기 합격예측)
```

### 최종 합격자 성적 반영 비율

| 채용 유형 | 필기 | 체력 | 면접 |
|---------|------|------|------|
| 공채 | **50%** | 25% | 25% |
| 경채 (체력시험 있음 — 구조·구급 등) | **50%** | 25% | 25% |
| 경채 (체력시험 없음 — 일부 분야) | **75%** | — | 25% |

> **서비스 범위**: 현재는 **필기시험 단계**만 다룹니다.
> 체력·면접 환산 기능은 미구현 (추후 확장 가능).

## 5. 필기 합격배수 ★ (핵심)

필기시험은 최종 선발인원의 **N배수**만큼만 합격시킵니다.
(나머지는 체력·면접을 보기 위한 후보)

### 5-1. 공채 합격배수

| 선발예정인원 | 필기 합격자 수 |
|----------|------------|
| 1~10명 | 선발인원 × **3배** |
| 11~20명 | 선발인원 × **2.5배** |
| 21~50명 | 선발인원 × **2배** |
| **51명 이상** | 선발인원 × **1.5배** |

**예시**: 공채 대구 남자 선발인원 48명
→ 21~50명 구간 → 합격배수 **2배**
→ 필기 합격자 = ceil(48 × 2.0) = **96명**

### 5-2. 경채 합격배수 (구조/소방학과/구급 공통)

| 선발예정인원 | 필기 합격자 수 |
|----------|------------|
| 1명 | **3명** |
| 2명 | **6명** |
| 3명 | **8명** |
| 4명 | **9명** |
| 5명 | **10명** |
| 6~50명 | 선발인원 × **1.8배** |
| **51명 이상** | 선발인원 × **1.5배** |

**예시**: 구급 경채 대구 남자 선발인원 35명
→ 6~50명 구간 → 합격배수 **1.8배**
→ 필기 합격자 = ceil(35 × 1.8) = **63명**

### 5-3. 동점자 처리

합격선 동점자는 선발 배수 초과하더라도 **전원 합격**.

> **코드**: `src/lib/prediction.ts` → `getPublicPassMultiple()`, `getCareerPassMultiple()`

## 6. 합격예측 피라미드 등급

서비스에서 5단계로 합격 가능성을 시각화합니다.

| 등급 | 기준 | 의미 |
|------|------|------|
| **확실권** | 내 순위 ≤ 모집인원 | 1배수 이내, 거의 확실 |
| **유력권** | 내 순위 ≤ 모집인원 × 1.2 | 안전권 내 |
| **가능권** | 내 순위 ≤ 합격배수 기준 인원 | 필기 합격 커트 이내 |
| **도전권** | 내 순위 > 합격배수 기준 인원 | 커트 초과, 동점자 여부에 달림 |
| **도전권 이하** | — | 합격 가능성 낮음 |

**배수 위치** = 내 순위 ÷ 모집인원
- 배수 ≤ 1.0 → 확실권
- 배수 ≤ 1.2 → 유력권
- 배수 ≤ 합격배수 → 가능권
- 배수 > 합격배수 → 도전권

> **코드**: `src/lib/prediction.ts` → `calculatePrediction()`

## 7. 가산점 체계

### 7-1. 필기 단계 가산점 (취업지원/의사상자)

| 종류 | 비율 | 제한 |
|------|------|------|
| 취업지원대상자 | 과목별 만점의 **5%** 또는 **10%** | 선발인원 30% 초과 불가 |
| 의사상자 등 | 과목별 만점의 **3%** 또는 **5%** | 선발인원 10% 초과 불가 |

- 두 종류 중 **유리한 1개만** 선택 (중복 불가)
- 과락 과목에는 가산점 미적용
- **공채**: 100점 × 비율 × 3과목에 각각 적용 → 최대 30점 (10%)
- **경채**: 100점 × 비율 × 2과목에 각각 적용 → 최대 20점 (10%)

**코드**: `src/lib/scoring.ts` → `BonusType` enum, `BONUS_RATE_BY_TYPE`

### 7-2. 자격증 가산점 (최종 환산 단계 — 현재 수집만)

| 가산점 | 해당 자격 |
|-------|---------|
| 0점 | 해당 없음 |
| 2점 | 소방 관련 자격 하위 등급 등 |
| 4점 | 소방 관련 자격 상위 등급 등 |
| 5점 | 최상위 자격 등 |

> **중요**: 자격증 가산점은 **필기 finalScore에 포함되지 않습니다**.
> DB의 `Submission.certificateBonus`에 저장하지만 순위 계산에 미사용.
> 향후 최종환산 기능 구현 시 활용.

## 8. 성별 분리 선발 — 핵심 규칙 ★★★

> **AI 협업자 필독**: 석차·1배수·합격배수·합격예측은 모두 아래 **성별 기준 모집단** 안에서만 계산합니다.

### 8-1. 유형별 성별 기준 요약

| `CAREER_RESCUE` | **구조** | 남자만 채용 | `recruitRescue` | 여자 접수 불가 / 단일 모집단 (성별 필터 불필요) |

| `CAREER_EMT` | **구급 여자** | 여자만 | `recruitEmtFemale` | 여자 모집단 내에서만 석차·1배수·합격예측 |

### 8-2. 유형별 상세 규칙

#### 공채 (PUBLIC) — 남녀 완전 분리

- 남자 응시자는 **해당 지역 공채 남자 응시자**끼리만 비교
- 여자 응시자는 **해당 지역 공채 여자 응시자**끼리만 비교
- 석차, 1배수 커트 점수, 합격배수, 합격예측 피라미드 → **각자 본인 성별 모집단 기준**
- 모집인원: 남자 → `recruitPublicMale`, 여자 → `recruitPublicFemale`
- 1배수 커트 = 본인 성별 모집인원 순위의 점수

```
예) 대구 공채 남자 65명 모집 → 남자 참여자 중 65위의 점수가 1배수 커트
예) 대구 공채 여자 11명 모집 → 여자 참여자 중 11위의 점수가 1배수 커트
```

---

#### 구조 경채 (CAREER_RESCUE) — 남자만 채용

- **여자 응시 불가** (접수 단계에서 차단, `getRescueMaleOnlyError` 함수로 검증)
- 남자 응시자 전체를 단일 모집단으로 처리 (성별 필터 없이 모든 구조 경채 참여자가 같은 모집단)
- 석차, 1배수 커트, 합격배수, 합격예측 → 해당 지역 구조 경채 전체 참여자 기준
- 모집인원: `recruitRescue` (단일 값)

```
예) 대구 구조 13명 모집 → 구조 참여자 전체 중 13위의 점수가 1배수 커트
```

---

#### 소방학과 경채 (CAREER_ACADEMIC) — 지역마다 다름 ⚠️

지역별로 `ExamRegionQuota` 테이블의 값을 확인해 방식을 결정합니다.

**케이스 A: 양성(통합) 채용** — `recruitAcademicCombined > 0`인 지역

- 남녀 구분 없이 **양성 응시자 전체**가 같은 모집단에서 경쟁
- 모집인원: `recruitAcademicCombined`
- 석차·1배수·합격예측 → 양성 전체 기준

```
예) 서울 소방학과 양성 4명 → 남녀 모두 같은 모집단, 4위 점수가 1배수 커트
```

**케이스 B: 남녀 분리 채용** — `recruitAcademicCombined = 0` 또는 `null`인 지역

- 남자는 **소방학과 남자 응시자**끼리만 비교, 모집인원: `recruitAcademicMale`
- 여자는 **소방학과 여자 응시자**끼리만 비교, 모집인원: `recruitAcademicFemale`
- 석차·1배수·합격예측 → 각자 본인 성별 모집단 기준

```
예) 인천 소방학과 남자 4명 / 여자 2명 분리 → 남자는 남자 4위, 여자는 여자 2위가 각각 1배수 커트
```

**코드 구현**: `buildPopulationWhere()` 함수가 `quota.recruitAcademicCombined`를 확인해 분기 처리

**양성(통합) 채용 지역의 전형 단계별 흐름**:

```
[필기] 과락(과목별 40% + 총점 60%) → 합격배수 통과
  → 남녀 구분 없이 같은 점수 기준으로 통합 경쟁, 고득점 순 필기 합격
      ↓
[체력] 기준 통과 (성별 불문 동일 기준)
      ↓
[면접] 점수 산출
      ↓
[최종] 필기 50% + 체력 25% + 면접 25% + 자격증 가산점
  → 최종점수 고득점 순으로 모집인원만큼 선발 (남녀 통합)
```

> 현재 서비스는 필기 단계만 예측. 자격증 가산점은 DB 저장만 하고 필기 순위에 미반영.

---

#### 구급 경채 (CAREER_EMT) — 남녀 완전 분리

- 남자 응시자는 **해당 지역 구급 남자 응시자**끼리만 비교
- 여자 응시자는 **해당 지역 구급 여자 응시자**끼리만 비교
- 석차, 1배수 커트, 합격배수, 합격예측 피라미드 → **각자 본인 성별 모집단 기준**
- 모집인원: 남자 → `recruitEmtMale`, 여자 → `recruitEmtFemale`

```
예) 대구 구급 남자 32명 / 여자 10명 분리
→ 남자: 남자 참여자 중 32위 점수가 1배수 커트
→ 여자: 여자 참여자 중 10위 점수가 1배수 커트
```

### 8-3. 모집단 결정 로직 (의사코드)

```typescript
// buildPopulationWhere() 에서 사용하는 성별 필터 로직
function buildGenderFilter(examType, gender, quota) {
  switch (examType) {
    case PUBLIC:
      return { gender }; // 반드시 본인 성별만

    case CAREER_RESCUE:
      return {}; // 성별 필터 없음 (남자만 접수 가능하므로)

    case CAREER_ACADEMIC:
      if (quota.recruitAcademicCombined > 0) {
        return {}; // 양성 통합 — 성별 필터 없음
      } else {
        return { gender }; // 남녀 분리 — 본인 성별만
      }

    case CAREER_EMT:
      return { gender }; // 반드시 본인 성별만
  }
}
```

### 8-4. 이 규칙이 적용되는 계산 목록

아래 계산은 **모두 위 모집단 안에서만** 이루어져야 합니다:

| 계산 항목 | 설명 |
|---------|------|
| **석차 (rank)** | 본인 성별 모집단 내 finalScore 내림차순 순위 |
| **1배수 커트 점수** | 본인 성별 모집인원 번째 사람의 점수 |
| **합격배수** | 본인 성별 모집인원을 기준으로 N배수 계산 |
| **합격예측 피라미드** | 확실/유력/가능/도전 등급 — 본인 성별 모집단 기준 |
| **경쟁자 목록** | 본인 성별 모집단의 타 참여자만 표시 |
| **평균점수/백분위** | 본인 성별 모집단 기준 통계 |
| **패스컷 히스토리** | *(예외)* 합산 분포로 계산 — 별도 설명 |

> **패스컷 히스토리 예외**: `src/lib/pass-cut.ts`에서 계산하는 과거 패스컷 이력은
> 성별 구분 없이 전체 합산 분포를 사용합니다 (성별별 데이터가 충분하지 않을 수 있어).
> 이 경우 모집인원도 남+여 합산값으로 사용합니다.

quota 확인 (ExamRegionQuota 테이블):

if (recruitAcademicCombined > 0) {
  → 양성 채용 지역

  → 여자: 모집인원 = recruitAcademicFemale, 여자 응시자끼리만 비교

## 9. 서비스 데이터 흐름

```
[관리자] 정답키 업로드 (/admin/answers)
    ↓
[사용자] OMR 답안 제출 (/exam/input)
    ↓
[시스템] 자동 채점
  - 과목별 원점수 = 정답수 × 배점
  - 과락 판정 (과목별 40% + 총점 60%)
  - 가산점 적용 → finalScore 저장
    ↓
[탭2] 내 성적 분석 (/exam/result)
  - 과목별 점수 차트
  - 원점수 / 백분위 / 석차 / 과락 여부
    ↓
[탭3] 합격예측 (/exam/prediction)
  - 동일 시험+지역+유형(+성별) 참여자 중 순위 계산
  - 배수 위치 = 내 순위 / 모집인원
  - 합격배수 대비 등급 판정 (확실/유력/가능/도전)
  - 경쟁자 목록 (이름 마스킹)
```

## 13. 수험번호 체계

소방 수험번호는 **총 10자리**입니다.

```
[P][P][P][G][TT][NNNN]
  ↑        ↑  ↑   ↑
prefix  성별 유형 순번
```

| 자리 | 의미 | 값 |
|------|------|-----|
| 1~3 | 지역·시험 prefix | 예: `211` |
| 4 | **성별** | 남=`1`, 여=`2` |
| 5~6 | **채용유형 코드** | 공채=`01`, 구급=`03`, 구조=`04`, 소방학과=`05` |
| 7~10 | **순번** | `0001`~ (4자리 0패딩) |

**예시**: `2111030152` → prefix `211` + 남자 `1` + 구급 `03` + 152번째
