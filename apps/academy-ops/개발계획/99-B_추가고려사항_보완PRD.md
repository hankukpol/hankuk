# 추가 고려사항 보완 PRD

**작성일**: 2026-03-13
**목적**: 기존 PRD에서 누락된 필수 기능과 운영 현실에서 반드시 필요한 항목을 추가 정의한다.

---

## 1. 법적 의무 사항 (학원법·개인정보보호법)

> 법적 근거가 있는 항목은 시스템 오픈 전 반드시 구현해야 한다.

### 1-1. 수강계약서 (학원법 §14 교부 의무)

학원은 수강계약 체결 시 수강계약서를 의무적으로 교부해야 한다.

```
수강 등록 완료 시 자동 생성

수 강 계 약 서
──────────────────────────────────────────────────
학원명:    ○○공무원학원  사업자번호: 000-00-00000
원장:      홍길동         전화:       02-0000-0000
주소:      서울시 ○○구 ○○로 000
──────────────────────────────────────────────────
수강생:    김수험  (수험번호: 2605001)
연락처:    010-1234-5678
강좌명:    2026 공채 종합반 52기
수강기간:  2026-03-01 ~ 2026-05-31
수강료:    600,000원
납부방법:  일시납 / 분할납부 2회
──────────────────────────────────────────────────
환불기준 (학원법 제18조 시행령 기준)
  수강 개시 전:           전액 환불
  수강기간 1/3 미경과:    납부금액의 2/3 환불
  수강기간 1/3~1/2 경과:  납부금액의 1/2 환불
  수강기간 1/2 초과:      환불 없음
──────────────────────────────────────────────────
위 사항에 동의하고 수강계약을 체결합니다.

  2026년  3월  1일

  수강생 서명: ________________
  학원 확인:  ________________  (직인)
──────────────────────────────────────────────────
```

**시스템 처리:**
- 수강 등록 완료 시 자동 생성 → PDF 출력
- 서명란: 출력 후 수기 서명 (Phase 1) / 태블릿 전자 서명 (Phase 2)
- 계약서 사본 DB 저장 (스캔 이미지 또는 PDF 첨부)

### 1-2. 수강료 초과징수 방지 (학원법 §15)

```
수강료 납부 등록 시 자동 검증:

강좌 정원: 60명  현재 납부 총액 상한선: 36,000,000원
학원 설정 수강료: 600,000원/인

납부 처리 전 체크:
  해당 학생의 동일 강좌 납부 이력 조회
  → 이미 납부 완료된 강좌에 중복 납부 시 경고 표시
  → 학원법 기준 초과 징수 금액 감지 시 원장 승인 필요
```

### 1-3. 개인정보 수집·이용 동의

```
학생 등록 화면 하단:

개인정보 수집·이용 동의
  ────────────────────────────────────────────────
  수집 항목: 이름, 연락처, 생년월일
  수집 목적: 수강 관리, 성적 통지, 출결 안내
  보유 기간: 수강 종료 후 3년
  ────────────────────────────────────────────────
  [ ☑ 필수 동의 ]  개인정보 수집·이용에 동의합니다.
  [ ☐ 선택 동의 ]  마케팅 목적 SMS 수신에 동의합니다.

동의 없이 등록 불가 (필수 항목)
동의 일시 기록 → 감사 로그 저장
```

## 2. 수강 워크플로우 강화

### 2-1. 반 정원 관리 + 대기자 명단

```
강좌 마스터 설정에 추가:
  정원: [ 60 ] 명
  대기자 허용: [ ● 허용  ○ 불허 ]
  최대 대기자: [ 10 ] 명

수강 등록 시:
  현재 등록: 58명 / 정원: 60명 → 정상 등록
  현재 등록: 60명 / 정원: 60명 → 대기자 등록 여부 확인

대기자 목록
  순번  이름     연락처        신청일
  1    이대기   010-1111-2222  2026-03-05
  2    박수험   010-3333-4444  2026-03-07

  [결원 발생 시 자동 SMS 안내] ← 설정 가능
```

**DB 추가 필드:**
```prisma
// CourseEnrollment에 추가
status  EnrollmentStatus  // ACTIVE, WAITING, CANCELLED, WITHDRAWN

enum EnrollmentStatus {
  ACTIVE     // 정상 수강
  WAITING    // 대기자
  CANCELLED  // 등록 취소
  WITHDRAWN  // 퇴원
}
```

### 2-2. 휴원 관리

공무원 시험 준비 중 군입대·취업 등으로 일시 휴원하는 경우가 자주 발생한다.

```
/admin/members/[id]/leave

휴원 신청 등록
══════════════════════════════════════════════════════
학생:       홍길동 (2605001)
강좌:       2026 공채 종합반

휴원 기간
  시작일: [ 2026-04-01 ]   종료일: [ 2026-05-31 ]
  기간:   61일

수강료 처리
  잔여 수강기간: 61일
  처리 방법:
  [ ● 복귀 시 기간 연장 ]
  [ ○ 잔여 수강료 환불 ]
  [ ○ 크레딧 적립 (다음 기수 사용) ]

사유: [ 군 입대 ▼ ]
비고: [                ]
══════════════════════════════════════════════════════
[신청 저장]  → 원장 승인 필요 (설정 가능)
```

**휴원 복귀 처리:**
```
복귀 등록
  복귀일: [ 2026-06-01 ]
  연장된 수강 종료일: 2026-07-31 (61일 자동 연장)
  수강 정보 자동 복원 (사물함·스터디룸 포함)
```

**DB:**
```prisma
model LeaveRecord {
  id           String      @id @default(cuid())
  examNumber   String
  courseEnrollmentId String
  startDate    DateTime
  endDate      DateTime
  returnDate   DateTime?
  leaveType    String      // MILITARY/JOB/PERSONAL/OTHER
  treatment    String      // EXTEND/REFUND/CREDIT
  approvedBy   String?
  approvedAt   DateTime?
  note         String?

  @@map("leave_records")
}
```

### 2-3. 수강 등록 시 교재 세트 자동 추천

```

추천 교재

  ┌─────────────────────────────────────────────┐

   (학생 정보 자동 입력, 세트 자동 선택)
```

## 4. 알림 시스템 강화 (카카오 알림톡)

### 4-1. 카카오 알림톡 연동

SMS보다 도달률 높고 비용 저렴 (건당 약 8~15원 vs SMS 20원).
카카오 비즈니스 채널 등록 후 알림톡 API 사용.

```
/admin/settings/notifications

알림 발송 채널 설정
  [ ● 카카오 알림톡 우선 (실패 시 SMS 폴백) ]
  [ ○ SMS만 사용 ]
  [ ○ 알림톡만 사용 ]

카카오 알림톡 설정
  비즈니스 채널 ID: [ @○○공무원학원 ]
  API 키:           [ xxxxxxxx ]
  발신 프로필:      [ ○○공무원학원 ]
```

### 4-2. 자동 알림 규칙 (상세)

## 12. 전체 우선순위 재정의

### 오픈 전 필수 (시스템 신뢰성)
- 수강계약서 자동 발행 (학원법)
- 개인정보 수집 동의 (법적 의무)
- 감사 로그 (분쟁 대비)
- 엑셀 일괄 등록 (초기 데이터 이관)

### Phase 1 운영 안정화
- 반 정원 + 대기자 관리
- 카카오 알림톡 연동
- 원장 대시보드
- 수강증 발급
- 영수증 이메일 발송

### Phase 2 편의성
- 휴원 관리
- 상담 전환율 통계
- 월간 운영 보고서
- 권한 세분화

### Phase 3 고도화
- 자동 백업
- 학생 사진 등록
- 수강 등록 시 교재 추천

## ⚠️ 멀티지점 고려사항 (2026-03-21 추가)

### 법적 의무 사항 — 지점별 독립 적용
- **수강계약서**: 지점별 학원명, 원장명, 주소, 계좌 정보 자동 삽입 (`academy_settings` 기반)
- **개인정보 동의**: 지점별 독립 (각 지점의 개인정보 처리방침 적용)
- **감사 로그 (AuditLog)**: `academyId` 포함 필수, 지점별 독립 조회

### 알림·통지 — 지점별 독립
- 카카오 알림톡: 지점별 발신키·채널ID 사용 (`academy_settings.kakaoSenderKey`, `kakaoChannelId`)
- SMS: 지점별 API 키 사용 (`academy_settings.smsApiKey`)
- 알림 템플릿: 지점명 변수 `{academyName}` 자동 치환

### 원장 대시보드
- 지점 원장 (DIRECTOR): 자신의 지점 KPI만 조회
- SUPER_ADMIN: `/admin/super/dashboard`에서 전 지점 통합 KPI 조회

### 보안·권한
- 미들웨어에서 `academyId` 기반 접근 제어 추가
- `/admin/super/*` 경로는 SUPER_ADMIN만 접근 가능 (별도 미들웨어 체크)
- 지점 간 데이터 교차 접근 시도 → 403 Forbidden 반환
