# 아침모의고사 성적·출결관리 — 운영 실수 대응 가이드

작성일: 2026-03-11

---

## 1. 학생 데이터 오류

### 1-1. 공채/경채 구분 오류
- **증상**: 경채 학생이 공채로, 또는 반대로 등록됨
- **해결**: `/admin/students` → 해당 학생 클릭 → 수정 → 시험 유형 변경 → 저장
- **참고**: 이미 입력된 성적은 세션(회차)에 연결되어 있으므로 성적 데이터는 그대로 유지됨. 재입력 불필요.

### 1-2. 기수(generation) / 반(class) 오류
- **증상**: 기수나 반이 잘못 입력됨
- **해결**: `/admin/students` → 해당 학생 클릭 → 수정 → 기수/반 변경 → 저장

### 1-3. 전화번호 오류
- **해결**: `/admin/students` → 해당 학생 클릭 → 수정 → 전화번호 변경 → 저장

### 1-4. 수험번호 오류 ⚠️ 현재 직접 해결 불가
- **증상**: 수험번호를 잘못 입력하거나 A·B 학생의 번호를 서로 바꿔 입력함
- **문제**: 수험번호는 DB의 Primary Key라 직접 변경 불가
- **현재 임시 해결 방법**:
  1. `/admin/students`에서 잘못된 학생 비활성화
  2. 올바른 수험번호로 신규 등록
  3. `/admin/scores/edit`에서 잘못 입력된 성적 삭제
  4. `/admin/scores/input`에서 올바른 수험번호로 성적 재입력
  5. 출결, 사유서, 상담 기록 등도 수동 재확인
- **→ 개발 예정**: 수험번호 이전 도구 (모든 연결 데이터 자동 이전)

### 1-5. 같은 학생 두 번 등록 (수험번호 다름)
- **증상**: 동일 학생이 다른 수험번호로 2개 계정이 생성됨
- **현재 임시 해결 방법**:
  1. 잘못된 수험번호 학생 비활성화
  2. 올바른 수험번호로 성적 등 수동 재확인
- **→ 개발 예정**: 학생 병합 도구

## 2. 성적 입력 오류

### 2-1. 특정 학생 점수 오류
- **해결**: `/admin/scores/edit` → 달력에서 날짜 선택 → 회차 선택 → 성적 조회 → 해당 학생 "수정" → 점수 변경 → 저장

### 2-2. 출결 유형 오류 (결석→정상, 정상→결석 등)
- **해결**: `/admin/scores/edit` → 해당 학생 "수정" → 응시 유형(attendType) 변경 → 저장
- **응시 유형 종류**: 정상(NORMAL), 라이브(LIVE), 인정(EXCUSED), 결석(ABSENT)

### 2-3. 회차 전체를 잘못된 날짜·회차에 업로드
- **해결**: `/admin/scores/edit` → 잘못 입력된 날짜 클릭 → 회차 선택 → **"회차 전체 삭제"** 버튼 → 올바른 날짜 회차에 재업로드
- **삭제 범위**: 점수, 출결, 문항 통계, 학생 답안, 오답 북마크 전부 삭제됨

### 2-4. 공채·경채 파일을 서로 바꿔 업로드
- **해결**: 2-3과 동일. 해당 회차 전체 삭제 후 올바른 파일로 재업로드

### 2-5. 성적 미입력 학생 확인
- **확인**: 대시보드의 성적 입력 현황 카드 또는 `/admin/notifications` → 성적 미입력 알림 기능

## 3. 기간·회차 오류

### 3-1. 회차 날짜(examDate) 오류
- **해결**: `/admin/periods` → 해당 기간 클릭 → 회차 목록에서 해당 회차 → 날짜 수정 → 저장
- **참고**: 날짜 변경 시 주간 상태 스냅샷 자동 재계산됨

### 3-2. 회차를 실수로 취소(isCancelled) 처리
- **해결**: `/admin/periods` → 해당 회차 → 취소 해제 → 저장 (토글 방식)

### 3-3. 잘못된 기간 활성화
- **해결**: 올바른 기간을 다시 활성화하면 자동으로 기존 기간이 비활성화됨 (동시에 1개만 활성 가능)

### 3-4. 회차 과목(subject) 오류 ⚠️ 현재 직접 해결 불가
- **→ 개발 예정**: updateSession에 subject 필드 추가

### 3-5. 회차 누락 (수동 추가 필요) ⚠️ 현재 직접 해결 불가
- **→ 개발 예정**: 개별 회차 수동 추가 기능

---

## 4. 수강생 등록 오류

### 4-1. 특정 학생을 잘못된 기간에 등록
- **해결**: `/admin/periods` → 해당 기간 → 수강생 탭 → 해당 학생 해제 → 올바른 기간에 다시 등록

### 4-2. 수백 명을 잘못된 기간에 일괄 등록 ⚠️ 현재 직접 해결 불가
- **현재**: 개별 해제만 가능 (100명이면 100번 클릭 필요)
- **→ 개발 예정**: 수강생 일괄 해제 기능

### 4-3. 미등록 학생 성적 입력 시도
- **자동 방지**: 수강생 미등록 시 성적 입력 시 매칭 실패로 자동 차단됨
- **해결**: 먼저 수강생 등록 후 성적 재입력

## 5. 사유서 오류

### 5-1. 사유서 잘못 승인 또는 잘못 반려
- **해결**: `/admin/absence-notes` → 해당 사유서 → "승인 취소" 버튼 → PENDING 상태로 복귀 → 재처리

### 5-2. 사유서 내용 수정 필요
- **해결**: `/admin/absence-notes` → 해당 사유서 → 수정

### 5-3. 사유서 삭제
- **해결**: `/admin/absence-notes` → 해당 사유서 → 삭제

### 5-4. 사유서 회차(날짜) 오류 ⚠️ 현재 직접 해결 불가
- **현재**: sessionId 변경 불가
- **임시 방법**: 기존 사유서 삭제 → 올바른 회차로 재등록
- **→ 개발 예정**: 사유서 회차 변경 기능

---

## 6. 알림 발송 오류

### 6-1. 동의 없는 학생 / 전화번호 없는 학생
- **자동 처리**: 시스템이 자동으로 스킵 (발송 안 됨)

### 6-2. 발송 후 취소 ⚠️ 영구 불가
- **주의**: 카카오 알림톡 / SMS는 한번 발송하면 회수 불가
- **예방이 최선**: 발송 전 대상자 목록 반드시 확인
- **→ 개발 예정**: 발송 전 대상자 미리보기 화면 강화 (이름, 전화번호, 상태 표시)

### 6-3. 결석 기록 오류로 인한 자동 탈락 알림 발송
- **위험 시나리오**: 결석 기록 오입력 → 자동 탈락 상태 변경 → 탈락 알림 자동 발송 → 취소 불가
- **예방**: 결석 기록 입력 전 반드시 재확인, 상태 변경 알림은 발송 전 목록 확인

## 7. 개발 예정 기능 목록

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| 긴급 | 수험번호 이전 도구 | 모든 연결 데이터(성적, 출결, 사유서 등)를 새 수험번호로 이전 |
| 높음 | 수강생 일괄 해제 | 잘못 등록된 수백 명을 한 번에 해제 |
| 높음 | 알림 발송 전 대상자 미리보기 강화 | 이름·전화번호·상태를 보여주는 확인 화면 |
| 높음 | 회차 과목(subject) 수정 | 잘못 설정된 과목 변경 가능 |
| 보통 | 개별 회차 수동 추가 | 누락된 회차나 임시 보충 시험 추가 |
| 보통 | 성적 bulk 업로드 시 이전 값 저장 | 덮어쓰기 전 기존 값을 감사 로그에 보존 |
| 보통 | 사유서 회차 변경 | 잘못된 날짜로 제출된 사유서의 회차 수정 |
| 낮음 | 학생 병합 도구 | 같은 학생이 두 수험번호로 등록된 경우 합치기 |
