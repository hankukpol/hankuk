## 2. 반드시 반영할 사항 (우선순위: 높음)

### 2.1 메인 페이지 TOP5 랭킹 테이블

**현재 문제**: 우리 메인 페이지(`/`)는 랜딩 페이지만 있고, 로그인 전 사용자에게 실시간 데이터를 보여주지 않음.

**반영 방안**:
- 메인 페이지 또는 `/exam/main` 페이지에 **TOP5 랭킹 테이블** 추가
- 로그인 없이 볼 수 있도록 공개 API로 제공
- 정렬 기준: 입력인원 순 / 1배수컷 순 / 경쟁률 순 (3가지 탭)

**필요 데이터 (이미 DB에 있는 것)**:
```
지역(Region) + 채용유형(ExamType) + 성별(Gender)별:
  - 선발인원: regions.recruitXxx 필드
  - 출원인원: regions.applicantXxx 필드 (관리자 입력)
  - 입력인원: submissions 테이블 COUNT
  - 1배수컷: PassCutSnapshot.oneMultipleCutScore
  - 확실권: PassCutSnapshot.sureMinScore
  - 평균: PassCutSnapshot.averageScore
```

**새로 만들 API**:
```
GET /api/top5?type=participants|oneMultipleCut|competitionRate
→ 상위 5개 지역×채용유형 조합 반환
```

**구현 위치**:
- API: `src/app/api/top5/route.ts` (새로 생성)
- UI: `src/app/page.tsx` 또는 `src/components/landing/Top5Ranking.tsx` (새로 생성)

---

### 2.2 경쟁률 표시

**현재 문제**: 우리는 "배수 위치"(순위/모집인원)로 개인의 위치를 보여주지만, 해당 모집단 전체의 경쟁 강도를 보여주는 **경쟁률**은 없음.

**반영 방안**:
```
경쟁률 = (과락 미포함 입력인원) / 선발인원

또는 소방단기처럼 보정:
경쟁률 = (과락 제외 유효 입력인원) / 선발인원
```

**표시 위치**:
- TOP5 랭킹 테이블
- 합격 컷 페이지(`/exam/prediction`) 내 정보 카드

---

### 2.3 합격예측 페이지에 1배수컷/확실권/평균 동시 표시

**현재 구현**: `PassCutSnapshot`에 `oneMultipleCutScore`, `sureMinScore`, `likelyMinScore`, `possibleMinScore`, `averageScore` 모두 저장하고 있음.

**소방단기 방식 참고**:
- 1배수컷: `oneMultipleCut` → 선발인원과 동일 순위 점수
- 확실권: `highestScore` → 상위 안정권
- 합격선: `passCut` → 배수 적용 후 커트라인

**반영할 표시 방식**:
```
내 점수: 210.00점
──────────────────
확실권 기준: 212.00점 이상
1배수컷:    204.00점
유력권 기준: 196.00점
가능권 기준: 188.00점
입력자 평균: 186.50점
```

→ 우리 프로젝트에 이미 데이터 구조가 있으므로 **UI 표시만 보강**하면 됨.

## 3. 참고하여 반영할 사항 (우선순위: 중간)

### 3.1 과목별 오답률/난이도 분석

**소방단기 구현**: 고득점 어려운 과목 TOP3 + 문항별 오답률 상위 3개

**반영 방안**:
- 채점 데이터(`user_answers`)에서 정답 대비 오답 비율 계산
- 공채/경채 소방학개론을 별도 집계 (소방단기도 이렇게 함)
- 표시 위치: 성적 분석 페이지(`/exam/result`) 하단 또는 메인 페이지

**필요 API**:
```
GET /api/stats/difficulty
→ 과목별 평균점수 + 문항별 오답률 TOP3
```

**구현 복잡도**: 중간 (user_answers 테이블에서 집계 쿼리)

---

### 3.2 실시간 참여인원 카운터

**소방단기**: 이용건수(PV) + 채점건수를 메인에 실시간 표시

**반영 방안**:
- 우리는 이미 관리자 대시보드(`/admin`)에 참여 통계가 있음
- **메인 페이지에도** 간단한 카운터 추가:
  ```
  현재 참여인원: 1,247명 | 오늘 신규: 342명
  ```
- 또는 TOP5 테이블 상단에 전체 참여자 수 표시

---

### 3.3 합격컷 발표 공지 자동 생성 개선

**소방단기**: "2026 소방직 1차 합격컷 공개" 공지를 시험 당일 게시

**우리 프로젝트**: 자동합격컷 시스템(`06_자동합격컷_시스템.md`)에 이미 공지 자동 생성 기능이 있음.
- 현재: "N차 합격컷 자동 발표 안내" 형식
- 개선: 공지 제목/내용 포맷을 더 사용자 친화적으로 수정

## 4. 참고만 할 사항 (우선순위: 낮음 / 향후 고려)

### 4.1 합격컷 문자 알림 신청

- 소방단기는 합격컷 공개 시 문자 알림 신청 기능 제공

### 4.3 이벤트/쿠폰 시스템

- 상업적 서비스 기능이므로 현재 범위 외

- 소방단기: 별도 채팅 탭

## 7. TOP5 API 설계안 (상세)

소방단기의 API 구조를 참고한 우리 프로젝트용 설계:

### 요청

```
GET /api/top5?type=participants&examId=1
GET /api/top5?type=oneMultipleCut&examId=1
GET /api/top5?type=competitionRate&examId=1
```

### 응답 (우리 스키마에 맞게 변환)

```typescript
interface Top5Item {
  rank: number;
  regionName: string;           // "대구"
  examType: string;             // "PUBLIC"
  examTypeName: string;         // "공채"
  gender: string;               // "MALE" | "FEMALE" | "ANY"
  genderName: string;           // "(남)" | "(여)" | "(양성)"
  displayName: string;          // "공채(남)" — 소방단기 jobName과 동일 형식
  recruitCount: number;         // 선발인원 65
  applicantCount: number | null; // 출원인원 745 (관리자 미입력 시 null)
  participantCount: number;     // 입력인원 214
  competitionRate: number;      // 경쟁률 2.57
  oneMultipleCut: number | null; // 1배수컷 204.00 (합격컷 미발표 시 null)
  averageScore: number | null;   // 평균 181.02
  sureMinScore: number | null;   // 확실권 216.00
}
```

### 데이터 소스

```sql
-- 입력인원 기준 TOP5
SELECT
  r.name as regionName,
  s.examType,
  s.gender,
  COUNT(*) as participantCount,
  -- PassCutSnapshot에서 최신 발표 데이터 JOIN
FROM submissions s
JOIN regions r ON s.regionId = r.id
WHERE s.examId = ? AND s.isDisqualified = false
GROUP BY r.id, s.examType, s.gender
ORDER BY participantCount DESC
LIMIT 5
```

## 8. 소방단기 데이터로 본 우리 서비스 목표치

### 대구 기준 목표

```
소방단기 대구 공채(남):
  출원 745명 중 214명 입력 → 참여율 28.7%

우리 서비스 대구 목표 (보수적 추정):
  대구·경북 중심 2천명 규모 사용자 대상
  소방단기보다 지역 밀착형이므로 대구·경북 참여율 더 높을 수 있음

  대구 공채(남) 65명 선발, 출원 745명 추정
  → 우리 서비스 참여자 최소 100명 이상 목표 (참여율 13%+)
  → 자동합격컷 1차 발표 조건: BALANCED 기준 참여율 30%
  → 약 (65 × 1.5배수) × 0.30 = 약 30명 이상이면 1차 발표 가능
```

### 자동합격컷 임계치와 소방단기 데이터 교차 검증

```
대구 공채(남): 선발 65명 → 합격배수 1.8배 → 필기합격 117명
BALANCED 1차 참여율 기준 30%: 117 × 0.30 = 약 35명

소방단기 실제 입력: 214명 (1배수컷 기준 모수의 약 183%)
→ 소방단기 수준이면 1차는 물론 4차까지 충분히 자동 발표 가능
→ 우리 서비스도 100명 이상 모이면 충분히 신뢰도 있는 합격컷 제공 가능
```

## 9. passCut vs oneMultipleCut 관계 심층 분석

소방단기 API에서 가장 흥미로운 점:

```
경기 공채(남):
  oneMultipleCut (1배수컷): 204.00  ← 319등의 점수
  passCut (합격선):         236.00  ← 이 값이 더 높음!
  highestScore (확실권):    212.00  ← 1배수컷보다 높고 passCut보다 낮음
```

**분석**:
- `oneMultipleCut < highestScore < passCut` 순서
- `passCut`은 단순 "합격 커트라인"이 아닌 다른 의미일 가능성:
  1. **합격배수 적용 후 안정권 상한선** (배수 내 최고점)
  2. **추정 실제 합격선** (과거 데이터 기반 보정)
  3. **확실권 상위 기준** (확실권 범위의 상한)

**우리 프로젝트에서의 대응**:
- `PassCutSnapshot.oneMultipleCutScore` = 1배수 커트라인 (소방단기의 `oneMultipleCut`)
- `PassCutSnapshot.sureMinScore` = 확실권 최저점 (소방단기의 `highestScore`에 해당)
- `PassCutSnapshot.likelyMinScore` = 유력권 최저점
- `PassCutSnapshot.possibleMinScore` = 가능권 최저점

→ 우리가 더 세분화된 등급 체계를 가지고 있어 소방단기보다 **더 정밀한 정보 제공 가능**.

## 10. 핵심 결론

### 소방단기가 잘하는 것 (벤치마킹 포인트)
1. **비로그인 사용자에게 실시간 데이터 노출** → 유입 유도 (TOP5 테이블)
2. **출원인원 대비 경쟁률** 표시 → 현실감 있는 경쟁 정보
3. **과목별 난이도 분석** → 채점 후 부가가치 콘텐츠
4. **시험 당일 즉시 운영** → 사전 입력 OPEN → 시험 직후 채점 → 바로 합격컷

### 우리가 잘하는 것 (차별화 포인트)
1. **4단계 자동 합격컷 시스템** → 관리 부담 최소화
2. **4등급 합격예측 피라미드** → 더 세분화된 합격 가능성 분석
3. **최신 기술 스택** → 반응형, 타입 안전, 확장성
4. **지역 밀착형** → 대구·경북 특화 (소방단기는 전국 서비스)

### 최종 결론
소방단기의 **TOP5 랭킹 + 경쟁률 + 오답률 분석**을 우리 프로젝트에 추가하면,
기존 강점(자동합격컷 + 4등급 피라미드)과 결합하여 **더 완성도 높은 서비스** 구현 가능.
