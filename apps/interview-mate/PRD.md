# PRD: 면접 스터디 조 편성 프로그램

## 1. 문서 목적

이 문서는 현재 구현된 `면접 스터디 조 편성 프로그램`의 제품 요구사항과 실제 동작 기준을 정리한다.
대상 코드는 Next.js 기반 로컬 웹앱이며, 경찰/소방 면접 준비생 명단을 입력받아 스터디 조를 자동 편성한다.

이번 버전의 핵심 변경은 기존 순차 배정 방식에서 `페널티 기반 스왑 최적화 알고리즘`으로 전환한 것이다.
또한 실제 목업 데이터 검증 결과를 반영해, `대구·경북`과 `나이 구간`은 고정 상한 방식이 아니라 `전체 분포 목표치와의 편차`를 줄이는 방식으로 보정되었다.

이후 추가된 주요 변경:
- 지역 분산을 `대구경북/타지역` 이진 분류에서 **각 지역별 독립 분산**으로 변경 (v2.1)
- 경찰/소방의 지역 분산 로직 분리 (v2.1)
- **짝수 배치 필수 직렬** 기능 추가 — 소수 직렬 단독 배치 방지 (v2.2)
- 강제 배정과 사전 편성 충돌 버그 수정 (v2.3)

## 3. 해결하려는 문제

기존 수동 편성 또는 단순 순차 배정 방식에서는 다음 문제가 반복됐다.

- 같은 지역 출신이 특정 조에 몰림
- 같은 성별 비율이 특정 조에 치우침
- 같은 나이대가 한 조에 몰림
- 특정 직렬이 과도하게 한 조에 몰림
- 조별 평균 성적 편차가 큼
- 이미 지정한 멤버나 반드시 넣어야 하는 직렬 배치를 유지하기 어려움

이 프로그램은 위 문제를 줄이기 위해 `하드 제약 + 소프트 페널티 최소화` 방식으로 조를 편성한다.

## 4. 제품 목표

1. 편성 시간을 줄인다.
2. 성별, 나이, 지역, 직렬, 성적 균형이 더 좋은 조를 만든다.
3. 사전 편성 및 강제 배정 규칙은 반드시 보존한다.
4. 결과를 즉시 화면에서 검토하고 엑셀로 내보낼 수 있게 한다.
5. 이전 편성 결과가 담긴 파일도 복원해서 다시 편성할 수 있게 한다.

## 11. 편성 알고리즘

현재 알고리즘은 `하드 제약 처리 -> 초기 배정 -> 반복 스왑 최적화 -> 검증/메트릭 산출`의 4단계로 구성된다.

### 11.1 Phase 0: 하드 제약 처리

#### 사전 편성

- `usePreAssignment = true`일 때만 적용
- `preAssignedGroup`이 지정된 멤버는 해당 조에 먼저 넣는다
- 사전 편성된 멤버는 `locked` 상태가 되며 스왑 대상에서 제외된다

#### 강제 배정

- 직렬별 `조당 N명` 규칙을 적용한다
- 사전 편성으로 이미 해당 직렬이 배치된 조는 부족분만 추가한다 (예: 조당 2명 규칙에서 사전 편성 1명이 있으면 1명만 추가)
- 아직 배정되지 않은 동일 직렬 멤버를 각 조에 순서대로 고정 배치한다
- 강제 배정된 멤버도 `locked` 상태가 되며 스왑 대상에서 제외된다
- 인원이 부족하면 가능한 범위까지만 배정하고 경고를 남긴다

### 11.2 Phase 1: 초기 배정

- 남/여 멤버를 분리한다
- 각 성별 내에서 직렬별 큐를 만든 뒤 `직렬 인터리브`로 섞는다
- 각 조의 최종 목표 인원 수와 목표 성비를 먼저 계산한다
- 이후 목표 성비에 더 부족한 성별부터 우선 배정한다
- 각 멤버는 `성별 목표치 + 조 인원 균형`을 함께 고려해 배정된다

### 11.3 Phase 2: 스왑 최적화

- 모든 조 쌍을 순회한다
- 잠금 멤버는 제외한다
- 같은 성별끼리만 스왑 후보가 된다
- 각 조 쌍 내부에서 `가장 페널티를 많이 줄이는 스왑`을 찾는다
- best-improving swap이 존재할 때만 실제 스왑을 실행한다
- 한 조 쌍 내부에서 더 이상 개선이 없을 때까지 반복한다
- 전체 iteration에서 개선이 없으면 종료한다
- 최대 반복 횟수는 `1000`

### 11.4 Phase 3: 검증 및 메트릭

- 조별 최소/최대 인원 위반 여부를 경고로 기록한다
- 총 페널티를 계산한다
- 항목별 페널티 분해값을 계산한다
- 반복 횟수, 스왑 횟수를 기록한다

## 12. 페널티 모델

조별 페널티는 다음 항목의 합으로 계산한다.

### 12.1 성별 혼합

- 전체 여성 비율 또는 수동 성비 기준을 목표치로 사용
- `abs(실제 여성 수 - 목표 여성 수) * 2.5`

### 12.2 나이 구간 분산

- 나이 구간:
  - A: 24세 이하
  - B: 25~27세
  - C: 28~30세
  - D: 31세 이상
- 나이 데이터가 있는 인원만 따로 집계해 전체 구간 비율을 계산한다
- 각 조는 `전체 나이 데이터 보유 비율`과 `A/B/C/D 비율`을 목표치로 가진다
- 페널티는 `조 내 연령 데이터 커버리지 편차 + 각 구간 인원 편차`의 합으로 계산한다
- 즉, `같은 구간 max 2` 같은 고정 상한이 아니라 `전체 명단의 연령 구조를 조마다 비슷하게 나누는 방식`이다

### 12.3 대구·경북 분산

- `지역` 값에 `대구` 또는 `경북`이 포함되면 대구·경북으로 간주
- 전체 명단의 `대구·경북 비율`을 조별 목표치로 사용한다
- 각 조의 실제 대구·경북 인원 수와 목표 인원 수의 차이를 계산한다
- 극단적인 쏠림을 더 강하게 억제하기 위해 `편차의 제곱`에 가중치를 곱한다
- 즉, `특정 상한을 넘었는지`보다 `조별 목표치에서 얼마나 멀어졌는지`를 더 중요하게 본다

### 12.4 직렬 분산

- 각 직렬별 전체 인원 기준 `조당 이상적 최대치`를 계산
- 해당 조에서 이상적 최대치를 초과한 인원만큼 페널티 부여
- `max(0, 직렬 인원 - idealMax) * 1.0`

### 12.5 성적 균등

- 성적이 있는 멤버만 계산에 포함
- 성적이 일부만 있는 경우, `조별 성적 데이터 보유 비율`도 함께 본다
- 전체 평균과 조 평균 간 차이를 표준편차로 정규화
- `성적 커버리지 편차 + (abs(조 평균 - 전체 평균) / 표준편차)`에 가중치를 곱한다
- 성적 데이터가 없거나 표준편차가 0이면 0

## 13. UI/기능 요구사항

### 13.1 메인 화면

- 경찰/소방 진입 화면 제공

### 13.3 명단 테이블

- 업로드된 멤버 전체 목록 표시
- 성별/직렬/지역/나이/성적 표시
- 사전 편성 조를 드롭다운으로 수정 가능

### 13.4 편성 설정

- 조당 최소/최대 인원
- 성비 자동/수동
- 사전 편성 적용 ON/OFF
- 최적화 기준 토글:
  - 성별 혼합
  - 나이 구간 분산
  - 대구·경북 분산
  - 직렬 분산
  - 성적 균등

토글 동작:

- ON: 기본 가중치 사용
- OFF: 해당 항목 가중치 0

조건부 노출:

- 나이 데이터가 있을 때만 `나이 구간 분산` 표시
- 성적 데이터가 있을 때만 `성적 균등` 표시

### 13.5 강제 배정 규칙

- 직렬 선택
- 조당 인원 입력
- 규칙 추가/삭제 가능

### 13.6 결과 화면

- 조별 카드 표시
- 조 인원 / 남녀 수 표시
- 대구·경북 인원 수 표시
- 직렬별 인원 배지 표시
- 나이 구간 분포 표시
  - 예: `A:2 B:3 C:2 D:1`
- 멤버별 표시:
  - 이름
  - 고정 여부
  - 성별
  - 직렬
  - 지역
  - 나이 + 구간
  - 성적

### 13.7 최적화 메트릭 표시

- 총 페널티
- 반복 횟수
- 스왑 횟수
- 항목별 페널티 분해

### 13.8 결과 내보내기

- 엑셀 파일로 저장
- 전체 요약 시트 + 조별 시트 생성
- 나이 값은 내보내지만, 나이 구간 라벨은 별도 컬럼으로 저장하지 않음

- 최대 인원 초과
- 강제 배정 대상 직렬 인원 부족
- 잘못된 조 인원 설정 입력값 보정
- 잘못된 사전 편성 조 번호

---

## 15. 현재 제한 사항

1. 편성은 브라우저 메모리 기반이며 서버 저장 기능이 없다.
2. 최적화는 same-gender swap만 허용하므로 성비 자체를 뒤집는 스왑은 하지 않는다.
3. 지역은 `대구/경북`과 `기타`의 2분류만 사용한다.
4. 나이 구간은 A/B/C/D 4분류를 사용하며, 목표치는 전체 데이터 비율 기준으로 자동 계산된다.
5. 가중치 수치를 직접 입력하는 UI는 없고, ON/OFF 토글만 제공한다.

---

## 16. 품질 기준

현재 코드 기준으로 다음을 만족해야 한다.

- 사전 편성 멤버는 결과에서도 지정 조를 유지한다.
- 강제 배정 멤버는 스왑 최적화 중 이동하지 않는다.
- 나이 데이터가 없으면 나이 페널티는 자동으로 0이 된다.
- 성적 데이터가 없으면 성적 페널티는 자동으로 0이 된다.
- `다시 섞기`를 누르면 다른 초기해에서 다시 최적화할 수 있다.
- 300명 기준 편성은 실사용 가능 속도로 완료되어야 한다.
- `mock-police-300.csv` 기본 설정 검증 시 대구·경북은 조당 `7~8명`, 여성은 `2~3명` 범위로 수렴해야 한다.
- `mock-fire-180.csv` 기본 설정 검증 시 대구·경북은 조당 `7~8명`, 여성은 조당 `3명`으로 수렴해야 한다.
- 현재 10인 조 기준에서 나이 구간은 `max 2`를 목표로 하지 않으며, 실제 검증 기준은 `지배적인 단일 구간이 4명 수준 이하로 수렴하는지`다.
