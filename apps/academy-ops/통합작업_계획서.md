#### 1-3. academyId 추가 필요 모델 목록 (~18개)

아래 모델은 현재 academyId가 없어서 멀티테넌시 지원을 위해 추가 필요:

1. `Notice` (line 544)
2. `Course` (line 769)
3. `Textbook` (line 791)
4. `Staff` (line 837)
5. `Instructor` (line 921)
6. `ComprehensiveCourseProduct` (line 959)
7. `Cohort` (line 978)
8. `SpecialLecture` (line 998)
9. `DailySettlement` (line 1346) — date @unique 제약 수정 필요 (academyId+date)
10. `Classroom` (line 1383)
11. `Locker` (line 1505) — zone+lockerNumber @unique → academyId 포함 필요
12. `StudyRoom` (line 1556)
13. `PointPolicy` (line 1809)
14. `PaymentLink` (line 1833)
15. `ConsultationProspect` (line 1906)
16. `ExamEvent` (line 1946)
17. `AbsencePolicy` (line 286) — name+absenceCategory @unique → academyId 포함 필요
18. `NotificationTemplate` (line 324) — type+channel @unique → academyId 포함 필요

※ Student, ExamPeriod, ExamSubject, Score, AbsenceNote, AdminUser, CourseEnrollment, Payment, ClassroomAttendanceLog, DiscountCode는 이미 academyId 있음

#### 1-4. 크론잡 수정 목록 (6개)

| 크론잡 | 경로 | 변경 내용 |
|--------|------|-----------|
| payment-links-expiry | `api/cron/payment-links-expiry` | 전체 학원 대상 (academyId 필터 불필요, 이미 글로벌) |
| enrollment-expiry | `api/cron/enrollment-expiry` | 학원별 순회 필요 |

| score-deadline | `api/cron/score-deadline` | 학원별 알림 발송 |
| weekly-report | `api/cron/weekly-report` | 학원별 리포트 생성 |
| payment-reminders | `api/cron/payment-reminders` | 학원별 알림 발송 |

#### 1-5. 데이터베이스 통합

1. 가장 큰/안정적인 Supabase 프로젝트를 메인으로 지정
2. Academy 레코드 생성 (fire, police, fire-interview 등)
3. 데이터 마이그레이션:
   - `cuid()` ID: 충돌 없음
   - `autoincrement` ID (~26개 모델): ID 오프셋 +1,000,000 적용
   - 모든 레코드에 academyId 스탬프
4. Supabase Auth 사용자 통합
5. Storage 통합 (absence-attachments 버킷)

## 4. 이미 수정된 파일 (되돌리기 또는 계속 진행)

### ✅ `prisma/schema.prisma`
- Academy 모델에 `hostname String? @unique` 및 `themeColor String?` 필드 추가

### ✅ `src/lib/academy.ts`
- `DEFAULT_ACADEMY_ID = 1` 제거
- `resolveAcademyByHostname()` 함수 추가 (hostname → academyId)
- `getAcademyBranding()` 함수 추가 (학원별 이름/테마색상)
- `ACADEMY_SELECT`에 hostname, themeColor 포함
- 모든 DEFAULT_ACADEMY_ID 참조를 `resolveAcademyByHostname()` 호출로 대체

### ✅ `src/lib/auth.ts`
- `DEFAULT_ACADEMY_ID` import 제거, `resolveAcademyByHostname` import 추가
- `getCurrentAdminContext()`에서 `DEFAULT_ACADEMY_ID` → `resolveAcademyByHostname()` 대체
