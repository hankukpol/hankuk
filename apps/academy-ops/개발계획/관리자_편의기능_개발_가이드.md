### A. 사유서 기능 강화 (2026-03-11 완료)

#### A-1. 사유서 회차(날짜) 변경
- **경로**: `/admin/absence-notes` → 사유서 클릭 → 드로어 하단 "회차 변경" 섹션
- **구현 파일**:
  - `src/lib/absence-notes/service.ts` — `changeAbsenceNoteSession()` 함수 추가
  - `src/app/api/absence-notes/[id]/route.ts` — `action: "changeSession"` 분기 추가
  - `src/components/absence-notes/absence-note-manager.tsx` — 드로어에 회차 변경 UI 추가
- **동작**:
  - 날짜 필터로 변경할 회차 선택 → "회차 변경" 버튼
  - APPROVED 상태인 경우 경고 후 진행 시 PENDING으로 자동 복귀 (재승인 필요)
  - 새 회차에 같은 학생 사유서가 이미 있으면 차단

#### A-2. 학생 일괄 사유서 등록 (단건→일괄 모드 전환)
- **경로**: `/admin/absence-notes` → 사유서 등록 섹션 → "일괄 등록" 탭
- **구현 파일**:
  - `src/lib/absence-notes/service.ts` — `bulkCreateAbsenceNotes()` 함수 추가
  - `src/app/api/absence-notes/bulk-create/route.ts` — 신규 API 엔드포인트
  - `src/components/absence-notes/absence-note-manager.tsx` — 일괄 등록 UI 추가
- **동작**:
  - 학생 선택 후 날짜 직접 선택 모드에서 여러 회차 체크박스 선택 → 일괄 등록
  - 이미 등록된 회차는 자동 건너뜀 (skipped)
  - 결과: "N건 등록 완료, M건 이미 존재(건너뜀)" 표시

#### A-3. 요일 반복 사유서 등록
- **경로**: `/admin/absence-notes` → 사유서 등록 → "일괄 등록" 탭 → "요일 반복" 서브탭
- **구현 파일**: `src/components/absence-notes/absence-note-manager.tsx` (UI만, API는 A-2와 공유)
- **동작**:
  - 요일 체크박스 (일월화수목금토) + 날짜 범위 입력
  - 조건에 맞는 회차 미리보기 자동 표시
  - 예: "매주 금요일, 3월 1일~3월 31일" → 해당하는 모든 금요일 회차 일괄 등록

## 개발 예정 기능

### B. 수험번호 이전 도구 (최우선)

**상황**: 수험번호를 잘못 입력하거나 A·B 학생 번호가 서로 바뀐 경우

**구현 계획**:
```
신규 페이지: /admin/students/transfer
신규 API:   POST /api/students/transfer
신규 서비스: src/lib/students/transfer.ts
```

**서비스 로직**:
```typescript
// transferStudentData(fromExamNumber, toExamNumber)
// 트랜잭션 내에서:
// 1. toExamNumber 학생이 이미 존재하면 차단 (또는 비활성화 후 진행)
// 2. Student 레코드 복사 (toExamNumber로)
// 3. 연결 데이터 일괄 업데이트:
//    - Score.examNumber
//    - PeriodEnrollment.examNumber
//    - AbsenceNote.examNumber
//    - CounselingRecord.examNumber
//    - PointLog.examNumber
//    - WeeklyStatusSnapshot.examNumber
// 4. 구 Student 비활성화
// 5. AuditLog에 이전 기록
```

**UI**:
- 구 수험번호 입력 → 학생 정보 조회
- 신 수험번호 입력
- 이전될 데이터 목록 미리보기 (성적 N건, 출결 N건, 사유서 N건 등)
- "이전 실행" 버튼 + 이중 확인

### D. 알림 발송 전 대상자 미리보기 강화

**상황**: 발송 후 취소 불가 → 발송 전 반드시 확인 필요

**현재 문제**: 발송 버튼 클릭 시 confirm 팝업만 있음

**구현 계획**:
```
수정 파일: src/components/notifications/notification-center.tsx
           → 발송 전 확인 모달 추가
```

**모달 내용**:
- 발송 대상자 목록 (이름, 전화번호, 현재 상태)
- 제외 대상 (동의 없음, 전화번호 없음)
- 발송 메시지 미리보기
- "N명에게 발송" 최종 확인 버튼

---

### E. 회차 과목(subject) 수정

**상황**: 회차 생성 시 과목을 잘못 설정한 경우

**현재**: `updateSession()`에 subject 필드 없음

**구현 계획**:
```
수정 파일: src/lib/periods/service.ts
           → updateSession() payload에 subject?: Subject 추가
수정 파일: src/app/api/periods/[id]/route.ts
           → subject 파싱 추가
수정 파일: src/components/periods/period-manager.tsx
           → 회차 수정 폼에 subject 드롭다운 추가
```

---

### F. 개별 회차 수동 추가

**상황**: 임시 보충 시험이나 회차 자동 생성 후 누락된 경우

**구현 계획**:
```
신규 API:   POST /api/periods/[id]/sessions
신규 서비스: createSession() in src/lib/periods/service.ts
수정 파일: src/components/periods/period-manager.tsx
           → "회차 추가" 버튼 + 날짜/주차/과목/시험유형 입력 폼
```

---

### G. 성적 bulk 업로드 시 이전 값 저장

**상황**: 파일 업로드로 덮어쓸 때 이전 성적이 AuditLog에 없음

**현재**: `applyParsedImport()`에서 `before: null`로 기록

**구현 계획**:
```
수정 파일: src/lib/scores/service.ts
           → applyParsedImport() 내 upsert 전에 기존 Score 조회
           → AuditLog.before에 기존 값 저장
```
