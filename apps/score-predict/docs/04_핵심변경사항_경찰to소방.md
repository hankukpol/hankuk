# 경찰 → 소방 변환 핵심 변경사항

> **AI 협업자 참고**: 이 문서는 `police/` 프로젝트에서 `fire/`로 변환하면서 실제로 수정된
> 파일과 변경 내용을 상세히 기술합니다. 코드 수정 시 반드시 확인하세요.

---

## 1. ExamType 변환

| 경찰 (`police/`) | 소방 (`fire/`) |
|----------------|--------------|
| `PUBLIC` (공채, 통합선발) | `PUBLIC` (공채, 남/여 **분리**선발) |
| `CAREER` (경행경채) | `CAREER_RESCUE` (구조, 남자만) |
| *(없음)* | `CAREER_ACADEMIC` (소방학과, 지역별 남/여/양성) |
| *(없음)* | `CAREER_EMT` (구급, 남/여 분리선발) |

> 경찰의 2종 → 소방의 4종으로 확장됨

## 2. ExamRegionQuota 필드명 변경

경찰과 소방은 DB 필드명이 완전히 다릅니다.

| 경찰 (OLD) | 소방 (NEW) | 비고 |
|-----------|-----------|------|
| `recruitCountPublicMale` | `recruitPublicMale` | |
| `recruitCountPublicFemale` | `recruitPublicFemale` | |
| `recruitCountCareerRescue` | `recruitRescue` | |
| `recruitCountCareerEmt` | `recruitEmtMale` / `recruitEmtFemale` | 남녀 분리됨 |
| *(없음)* | `recruitAcademicMale` | 소방학과 신규 |
| *(없음)* | `recruitAcademicFemale` | 소방학과 신규 |
| *(없음)* | `recruitAcademicCombined` | 소방학과 양성 신규 |
| `applicantCountPublicMale` | `applicantPublicMale` | |
| `applicantCountPublicFemale` | `applicantPublicFemale` | |
| `applicantCountCareerRescue` | `applicantRescue` | |
| `applicantCountCareerEmt` | `applicantEmtMale` / `applicantEmtFemale` | 남녀 분리됨 |

```typescript
// 경찰

```

### 3-2. 총점 과락 추가 (소방 전용)

```typescript
// 경찰에는 없음, 소방에만 있음
const totalMaxScore = subjects.reduce((sum, s) => sum + s.maxScore, 0);
const hasTotalCutoff = totalScore < totalMaxScore * TOTAL_CUTOFF_RATE; // 60%
```

### 3-3. toRankGroupKey 변경

```typescript
// 경찰: 성별 구분 없음 (통합선발)
// 소방: CAREER_RESCUE 제외 모두 성별 포함
function toRankGroupKey(regionId, examType, gender) {
  if (examType === ExamType.CAREER_RESCUE) return `${regionId}:${examType}`;
  return gender ? `${regionId}:${examType}:${gender}` : `${regionId}:${examType}`;
}
```

```typescript
// 경찰
PUBLIC → "공채"

PUBLIC → "공채(남)" / "공채(여)" / "공채" (성별에 따라)

```

### 4-2. getRecruitCount 변경 (지역 quota 기반)

```typescript
// 경찰: 단순 조회
// 소방: CAREER_ACADEMIC은 양성 여부 확인 후 분기
case ExamType.CAREER_ACADEMIC:
  if (quota.recruitAcademicCombined > 0) return quota.recruitAcademicCombined;
  return gender === MALE ? quota.recruitAcademicMale : quota.recruitAcademicFemale;
```

### 4-3. 합격배수 테이블 분리

```typescript
// 경찰: 단일 테이블 (1.5~1.8배)
// 소방: 공채/경채 별도 테이블
getPublicPassMultiple(recruitCount)  // 1.5~3배
getCareerPassMultiple(recruitCount)  // 1.5~1.8배 + 5명 이하 특수
```

### 4-4. buildPopulationWhere 변경

```typescript
// 경찰: 단순 { examId, regionId, examType }
// 소방: CAREER_ACADEMIC은 quota 확인 후 성별 필터 결정
function buildPopulationWhere(submission, quota?) {
  if (examType === CAREER_ACADEMIC && quota?.recruitAcademicCombined > 0) {
    // 양성 지역: 성별 무관 전체 경쟁
    return { examId, regionId, examType };
  }
  // 나머지: 성별 포함
  return { examId, regionId, examType, gender };
}
```

## 6. 현재 상태 / 운영 주의사항

### 6-1. 서버 실행 이슈
- `tailwindcss` 모듈 해석이 부모 디렉토리(`합격예측 프로그램/`)를 참조하는 Turbopack 버그가 간헐 발생
- 회피 방법: Turbopack 비활성화 (`next dev --no-turbo`) 또는 `scripts/dev-single-process.cjs` 사용

### 6-2. 소방학과 CAREER_ACADEMIC 양성 지역 순위 처리 (반영 완료)
- `prediction.ts`의 `buildPopulationWhere()`에서 양성 지역(`recruitAcademicCombined > 0`)은 성별 필터 제거
- `scoring.ts` 재채점 랭크맵도 동일 규칙(양성 지역 성별 통합) 적용
- `api/result` 집계 SQL도 동일 모집단 규칙으로 정렬되어 결과/예측 순위 기준 일치

### 6-3. 자격증 가산점 UI (반영 완료)
- `/exam/input`에 자격증 가산점 라디오(0/2/4/5점) 추가
- `Submission.certificateBonus` 저장/수정/조회 API 연결 완료
- 자격증 가산점은 현재 규정대로 필기 `finalScore`에는 미반영(최종 환산 단계용 데이터)

### 6-4. Vercel 배포 (반영 완료)
- `fire/` 전용 Vercel 프로젝트 생성 및 로컬 링크 완료 (`.vercel/project.json` 기준)
- 남은 작업: 프로젝트 환경변수 등록 (아래 6-5 포함)

### 6-5. SUPABASE_SERVICE_ROLE_KEY 미설정 (운영 잔여)
- 현재 `.env.local`의 `SUPABASE_SERVICE_ROLE_KEY=""` 빈 값
- 파일 업로드(배너 이미지 등) 기능 사용 시 Supabase Settings → API에서 서비스 역할 키 입력 필요
- 동일 값을 Vercel `fire` 프로젝트 환경변수에도 등록 필요

### 6-6. 브랜딩 (반영 완료)
- 기본 팔레트를 소방 레드 계열(`red-600` 중심)로 교체
- OMR 주요 인터랙션 색상(선택/포커스/호버) 레드 계열 반영
