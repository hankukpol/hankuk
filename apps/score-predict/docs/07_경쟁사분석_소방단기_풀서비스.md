## 4. 합격예측 핵심 데이터 모델

### 4.1 API 응답 스키마 (TOP5)

```json
{
  "number": null,
  "jobName": "공채(남)",
  "nowSelect": "319",
  "areaName": "경기",
  "areaDetailName": "-",
  "jobTypeName": "선택없음",
  "nowApply": "2793명",
  "competitionRate": "2.45",
  "cnt": "951",
  "passCut": "236.00",
  "averageScore": "186.52",
  "highestScore": "212.00",
  "oneMultipleCut": "204.00"
}
```

### 4.2 핵심 점수 필드 관계

```
highestScore (확실권) > passCut (합격선) > oneMultipleCut (1배수컷) > averageScore (평균)

예시 (경기 공채남):
  확실권:   212.00  ← 이 이상이면 합격 확실
  합격선:   236.00  ← 배수 적용 후 합격 커트라인 (※ UI 미노출, API에만 존재)
  1배수컷: 204.00  ← 선발인원과 동일 순위의 점수
  평균:     186.50  ← 입력자 전체 평균
```

> **주의**: `passCut`(236) > `highestScore`(212)인 케이스가 있어서,
> `passCut`은 실제 "합격 배수 적용 후 커트라인"이 아니라
> **합격배수 내 최저점** 또는 **별도 산출 기준**일 가능성이 높음.
> 또는 `highestScore`가 "확실권 기준점"이고, `passCut`은 "배수 적용 합격선"일 수 있음.

### 4.3 채용구분(jobName) 확인된 목록

| 채용구분 | 성별 구분 | 비고 |
|---------|---------|------|
| **공채(남)** | O | 남녀 분리 선발 |
| **공채(여)** | O | 남녀 분리 선발 |
| **구급(남)** | O | 경채 |
| **구급(여)** | O | 경채 |
| **구조(남)** | O | 경채 (남자만 선발) |
| **관련학과(남)** | O | 경채 - 소방관련학과 |
| **관련학과(양성)** | 양성 | 성별 무관 선발 |

> 우리 프로젝트의 ExamType과 거의 1:1 대응:
> - 공채(남/여) → `PUBLIC` + gender
> - 구조(남) → `CAREER_RESCUE`
> - 관련학과(남/양성) → `CAREER_ACADEMIC` + gender
> - 구급(남/여) → `CAREER_EMT` + gender

### 4.4 경쟁률 계산 방식 (추정)

단순 `입력인원/선발인원`이 아닌 **보정된 경쟁률** 사용:

```
경기 공채(남): 951 / 319 = 2.98 → 실제 표시: 2.45
```

가능한 보정 방법:
1. **과락자 제외** 후 경쟁률 계산
2. **출원인원 대비** 경쟁률 (2793 / 319×α)
3. 별도 통계 모델 적용

→ 과락자 제외가 가장 유력. 과락자를 빼면 유효 경쟁자가 줄어 경쟁률이 낮아짐.
