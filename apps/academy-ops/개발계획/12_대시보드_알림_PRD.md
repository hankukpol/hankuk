## 2. 카카오 알림톡

> 비즈니스 채널 개설 완료. 중요 이벤트 발생 시 카카오 알림톡으로 자동 발송.
> 발송 실패 시 SMS로 자동 폴백.

### 2-1. 알림 발송 이벤트 목록

> **확정 (2026-03-13 인터뷰)**: 현재 설정 그대로 운영. 🔴 필수 항목은 기본 ON, 나머지는 관리자 설정에서 ON/OFF.

| 이벤트 | 수신자 | 발송 시점 | 우선순위 |
|---|---|---|---|
| **수강 등록 완료** | 학생(보호자) | 수강 등록 즉시 | 🔴 필수 (기본 ON) |
| **수납 완료** | 학생(보호자) | 결제 완료 즉시 | 🔴 필수 (기본 ON) |
| **수강계약서 발행** | 학생(보호자) | 계약서 생성 즉시 | 🔴 필수 (기본 ON) |
| **모바일 수강증 발급** | 학생 | 수강증 발급 즉시 | 🔴 필수 (기본 ON) |
| **미납 독촉** | 학생(보호자) | 납부 기한 D+1, D+7, D+14 | 🔴 필수 (기본 ON) |
| **출결 경고** | 학생(보호자) | 경고 처리 즉시 | 🔴 필수 (기본 ON) |
| **성적 등록** | 학생 | 회차 전체 성적 입력 완료 후 | 🟡 권장 (기본 OFF) |
| **사유서 처리 결과** | 학생 | 승인/반려 즉시 | 🟡 권장 (기본 OFF) |
| **상담 예약 확인** | 학생 | 상담 예약 등록 시 | 🟡 권장 (기본 OFF) |
| **공지사항** | 대상 수강생 | 관리자 수동 발송 | 🟢 선택 |
| **합격 축하** | 합격자 | 합격자 등록 시 | 🟢 선택 |
| **환불 처리 완료** | 학생(보호자) | 환불 승인 즉시 | 🟡 권장 (기본 OFF) |

### 2-5. DB 모델

```prisma
model NotificationLog {
  id            String    @id @default(cuid())
  examNumber    String?   // 학생 수신자 (null이면 비학생)
  recipientName String
  recipientMobile String
  channel       NotifChannel   // KAKAO / SMS / EMAIL
  eventType     NotifEventType
  templateId    String?   // 카카오 알림톡 템플릿 ID
  content       String    // 발송 내용 스냅샷
  status        NotifStatus    // SENT / FAILED / FALLBACK
  failReason    String?
  sentAt        DateTime  @default(now())
  @@map("notification_logs")
}

enum NotifChannel { KAKAO SMS EMAIL }
enum NotifStatus  { SENT FAILED FALLBACK }
enum NotifEventType {
  ENROLLMENT_COMPLETE
  PAYMENT_COMPLETE
  CONTRACT_ISSUED
  STUDENT_CARD_ISSUED
  PAYMENT_OVERDUE
  ATTENDANCE_WARNING
  SCORE_PUBLISHED
  ABSENCE_NOTE_RESULT
  CONSULTATION_REMINDER
  GENERAL_NOTICE
  PASS_CONGRATULATIONS
  REFUND_COMPLETE
}
```

## 3. 학생 포털

> 학생이 스마트폰으로 본인의 수강 정보·성적·출결·수납내역을 조회하는 포털.
> 별도 앱 없이 **모바일 웹(PWA)** 형태로 제공.

### 3-1. 학생 포털 접근

```
URL: https://portal.academy.com
접근 방식: 학번 + 생년월일 로그인 (또는 카카오 간편 로그인)
```

