## 1. ExamType Enum

```prisma
enum ExamType {
  PUBLIC           // 공채 (남/여 분리 선발)
  CAREER_RESCUE    // 구조 경채 (남자만)
  CAREER_ACADEMIC  // 소방관련학과 경채 (남/여/양성 — 지역마다 다름)
  CAREER_EMT       // 구급 경채 (남/여 분리 선발)
}
```

> **경찰과 차이**: 경찰은 `PUBLIC` / `CAREER`(경행경채) 2종.
> 소방은 **4종**이며 각각 성별 선발 방식이 다름.

## 2. 핵심 테이블

### ExamRegionQuota — 지역별 모집인원 (소방 전용 구조)

```prisma
model ExamRegionQuota {
  id                     Int  @id @default(autoincrement())
  examId                 Int
  regionId               Int

  // 공채 (남/여 분리 선발)
  recruitPublicMale      Int  @default(0)
  recruitPublicFemale    Int  @default(0)

  // 구조 경채 (남자만 — 단일 필드)
  recruitRescue          Int  @default(0)

  // 소방학과 경채 (지역마다 남/여/양성 방식이 다름)
  recruitAcademicMale    Int  @default(0)  // 남자 모집
  recruitAcademicFemale  Int  @default(0)  // 여자 모집
  recruitAcademicCombined Int @default(0)  // 양성(통합) 모집

  // 구급 경채 (남/여 분리 선발)
  recruitEmtMale         Int  @default(0)
  recruitEmtFemale       Int  @default(0)

  // 실제 접수인원 (선택 — 관리자가 나중에 입력)
  applicantPublicMale       Int?
  applicantPublicFemale     Int?
  applicantRescue           Int?
  applicantAcademicMale     Int?
  applicantAcademicFemale   Int?
  applicantAcademicCombined Int?
  applicantEmtMale          Int?
  applicantEmtFemale        Int?

  // 응시번호 범위
  examNumberStart        String?
  examNumberEnd          String?

  @@unique([examId, regionId])
}
```

> ⚠️ **경찰 프로젝트와 필드명 완전히 다름**
> 경찰의 `recruitCountPublicMale` → 소방의 `recruitPublicMale`
> 경찰의 `recruitCountCareerRescue` → 소방의 `recruitRescue`

### Submission — 제출 답안

```prisma
model Submission {
  id               Int       @id @default(autoincrement())
  examId           Int
  userId           Int
  regionId         Int
  examType         ExamType  // PUBLIC / CAREER_RESCUE / CAREER_ACADEMIC / CAREER_EMT
  gender           Gender    // MALE / FEMALE (CAREER_RESCUE는 항상 MALE)
  examNumber       String
  totalScore       Float     // 원점수 합계
  bonusType        BonusType @default(NONE)  // 취업지원/의사상자 가산 유형
  bonusRate        Float     @default(0)    // 실제 적용된 가산 비율
  certificateBonus Float     @default(0)   // 자격증 가산점 (0/2/4/5) — 필기에 미적용
  finalScore       Float     // 최종 점수 (총점 + 취업지원/의사상자 가산)
  // ... 기타 필드
}
```

> **certificateBonus 주의**: 자격증 가산점은 `finalScore`에 포함되지 않습니다.
> 수집만 하며, 최종 합격자 환산 시 별도 계산이 필요합니다.

## 4. 과목(Subject) 데이터 구조

Seed에서 ExamType별로 Subject를 등록합니다:

```typescript
// PUBLIC (공채) — 3과목
{ name: "소방학개론",   questionCount: 25, pointPerQuestion: 4,   maxScore: 100, examType: "PUBLIC" }
{ name: "소방관계법규", questionCount: 25, pointPerQuestion: 4,   maxScore: 100, examType: "PUBLIC" }
{ name: "행정법총론",   questionCount: 25, pointPerQuestion: 4,   maxScore: 100, examType: "PUBLIC" }

// CAREER_RESCUE (구조) — 2과목
{ name: "소방학개론",   questionCount: 25, pointPerQuestion: 4,   maxScore: 100, examType: "CAREER_RESCUE" }
{ name: "소방관계법규", questionCount: 40, pointPerQuestion: 2.5, maxScore: 100, examType: "CAREER_RESCUE" }

// CAREER_ACADEMIC (소방학과) — 2과목 (구조와 동일)
{ name: "소방학개론",   questionCount: 25, pointPerQuestion: 4,   maxScore: 100, examType: "CAREER_ACADEMIC" }
{ name: "소방관계법규", questionCount: 40, pointPerQuestion: 2.5, maxScore: 100, examType: "CAREER_ACADEMIC" }

// CAREER_EMT (구급) — 2과목
{ name: "소방학개론",       questionCount: 25, pointPerQuestion: 4,   maxScore: 100, examType: "CAREER_EMT" }
{ name: "응급처치학개론",   questionCount: 40, pointPerQuestion: 2.5, maxScore: 100, examType: "CAREER_EMT" }
```

> `(name, examType)` 조합으로 UNIQUE 제약 → 과목명이 같아도 ExamType이 다르면 별도 Subject

## 5. 순위 산출 모집단 분리 규칙

`src/lib/scoring.ts`의 `toRankGroupKey()`:

```typescript
function toRankGroupKey(regionId, examType, gender) {
  if (examType === ExamType.CAREER_RESCUE) {
    // 구조: 남자만이므로 성별 키 불필요
    return `${regionId}:${examType}`;
  }
  // 공채/소방학과/구급: 성별 포함
  if (gender) return `${regionId}:${examType}:${gender}`;
  return `${regionId}:${examType}`;
}
```

`src/lib/prediction.ts`의 `buildPopulationWhere()`:

```typescript
// CAREER_ACADEMIC — 지역 quota 확인 후 분기
if (examType === ExamType.CAREER_ACADEMIC) {
  if (quota.recruitAcademicCombined > 0) {
    // 양성 채용 지역: 성별 필터 없음
    return { examId, regionId, examType };
  }
  // 남녀 분리 채용 지역: 성별 필터 적용
  return { examId, regionId, examType, gender };
}
```

## 6. DB 초기화 (Seed)

시드 파일: `prisma/seed.ts`

실행 방법:
```bash
# 일반 실행 (Transaction pooler - 실패 가능)
npx prisma db seed

# 권장: Session pooler URL로 직접 실행
DATABASE_URL="postgresql://postgres.iqhkmcxeuwueiqopkwfd:..." npx prisma db seed
```

시드 내용:
1. 관리자 계정 생성 (`.env.local`의 ADMIN_PHONE/ADMIN_PASSWORD 사용)
2. 2026년 소방 시험 생성
3. 18개 지역 생성
4. 지역별 모집인원 입력 (ExamRegionQuota)
5. 4종 ExamType별 과목 등록 (Subject)
