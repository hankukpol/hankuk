# PRD: 시스템 설정 센터 — 마스터 데이터 관리

**작성일**: 2026-03-13
**위치**: 상단 탭 → [시스템] → 좌측 사이드바 → 각 설정 항목
**목적**: 전체 기능에서 사용되는 기준 데이터(마스터 데이터)를 한 곳에서 등록·수정·삭제한다.

---

## 설계 원칙

> **"기능을 쓰기 전에 여기서 먼저 세팅한다"**
>
> - 강좌를 개설하기 전 → 강좌 마스터 등록
> - 스터디룸 예약을 받기 전 → 스터디룸 목록 등록
> - 사물함 배정하기 전 → 사물함 구역·번호 등록
> - 교재 대여하기 전 → 교재 목록 등록
> - 카카오 출결 쓰기 전 → 출석 키워드 등록

---

## 설정 센터 URL 구조

```
/admin/settings/
├── academy              학원 기본 정보
├── courses              강좌 마스터
├── subjects             과목 설정
├── study-rooms          스터디룸 목록
├── lockers              사물함 구역·배치
├── study-hall           자습실 설정
├── books                교재·교구 목록
├── civil-exams          공무원 시험 일정
├── payment-policies     수납 정책 (할인·쿠폰)
├── point-policies       포인트 적립·사용 정책
├── attendance-keywords  출결 키워드 (카카오)

└── data                 데이터 내보내기·이관
```

## 설정 그룹 1: 학원 기본 정보 (`/admin/settings/academy`)

모든 출력 문서(수강내역 확인서, 영수증 등)에 자동 삽입되는 정보.

| 항목 | 필드 | 비고 |
|---|---|---|
| 학원명 | text | 예: ○○공무원학원 |
| 원장 성명 | text | |
| 사업자등록번호 | text | 000-00-00000 형식 자동 포맷 |
| 학원 등록번호 | text | 예: 제 00-0000호 |
| 주소 | text | |
| 대표 전화 | text | |
| 학원 로고 | image upload | 출력 문서 상단 삽입 |
| 원장 직인 | image upload | 서명란에 삽입 |
| 계좌 정보 | text×3 | 은행명, 계좌번호, 예금주 (수납 안내용) |

## 설정 그룹 3: 과목 설정 (`/admin/settings/subjects`)

## 구현 시 핵심 원칙

1. **각 마스터 데이터는 CRUD 완비**: 목록 조회 + 등록 + 수정 + 삭제(또는 비활성화)

3. **변경 이력 자동 기록**: 설정 변경 시 AuditLog에 자동 기록 (누가, 무엇을, 언제 바꿨는지)
4. **원장(DIRECTOR) 전용**: 모든 설정 페이지는 DIRECTOR 권한만 접근 가능
5. **즉시 반영**: 설정 저장 즉시 관련 기능에 반영 (별도 '적용' 버튼 없음)
