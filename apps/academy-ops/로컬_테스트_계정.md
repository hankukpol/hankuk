# 로컬 테스트 계정

로컬 목업 서버 실행:

```bat
run-local-mock-dev.bat
```

포트를 지정하려면:

```bat
run-local-mock-dev.bat 3024
```

기본 URL:

- 관리자: `http://127.0.0.1:3000/admin`
- 관리자 로그인 안내: `http://127.0.0.1:3000/login`
- 학생 포털 로그인: `http://127.0.0.1:3000/student/login`

현재 목업 관리자 계정:

- 이름: `로컬 관리자`
- 이메일: `local-admin@morningmock.local`
- 권한: `SUPER_ADMIN`
- 로그인 방식: 로컬 목업 모드에서는 `/admin` 직접 진입

현재 목업 학생 계정:

- `홍길동`
  - 학번: `2501001`
  - 생년월일 6자리: `000115`
  - 반: `52기 공채 종합반`
- `김경채`
  - 학번: `2501002`
  - 생년월일 6자리: `010225`
  - 반: `52기 경채 종합반`

계정 데이터 소스:

- 시드 스크립트: `web/scripts/seed-local-mock.ts`
- 실행 배치 파일: `run-local-mock-dev.bat`

참고:

- 로컬 목업 모드에서는 Supabase 관리자 로그인을 사용하지 않습니다.
- 학생 포털은 학번 + 생년월일 6자리로 로그인합니다.
- `run-local-mock-dev.bat` 실행 시 현재 스키마를 로컬 Prisma DB에 맞춘 뒤 목업 데이터를 다시 넣습니다.
